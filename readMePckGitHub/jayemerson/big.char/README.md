    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-bigchar" class="anchor" href="#bigchar" aria-hidden="true"><span class="octicon octicon-link"></span></a>big.char</h1>

<p>Extends bigmemory's big.matrix to support big vectors of character strings.
Current package build status via use of
<a href="https://travis-ci.org/jayemerson/big.char">Travis CI</a>:
<a href="https://travis-ci.org/jayemerson/big.char"><img src="https://camo.githubusercontent.com/b0bff8322727bc96f7c7b9a458741d1d063bed96/68747470733a2f2f7472617669732d63692e6f72672f6a6179656d6572736f6e2f6269672e636861722e7376673f6272616e63683d6d6173746572" data-canonical-src="https://travis-ci.org/jayemerson/big.char.svg?branch=master" style="max-width:100%;"></a></p>

<h2>
<a id="user-content-description" class="anchor" href="#description" aria-hidden="true"><span class="octicon octicon-link"></span></a>Description</h2>

<p>This package extends bigmemory's big.matrix objects
to support big vectors of character strings.  The only constraint
is the user-specified maximum character length which must be
satisfied for every string in the vector.  Shorter strings are
padded with NAs, consuming (and wasting) space, but this is
unavoidable given the use of memory-mapped files.</p>

<h2>
<a id="user-content-note-under-development" class="anchor" href="#note-under-development" aria-hidden="true"><span class="octicon octicon-link"></span></a>NOTE: under development!</h2>

<p>This package is very much under development.  The basic functionality
seems fairly solid.  The implementation may not be as efficient as
it could be.  Not all usage errors may be trapped or handled
in a friendly or appropriate manner.
For example, if a <code>big.char</code> vector, <code>x</code> has length 5, then
<code>x[6]</code> generates an error (this would not be the case with a regular
vector of strings).  Please feel free to email me with problems,
questions, or requests.</p>

<p>Known issues to be resolved:</p>

<ul class="task-list">
<li>index recycling (bug in bigmemory, fix there)</li>
<li>indices out of bounds (needs fixing in big.char for consistency
with R's default behavior/behaviour)</li>
</ul>

<h2>
<a id="user-content-getting-started" class="anchor" href="#getting-started" aria-hidden="true"><span class="octicon octicon-link"></span></a>Getting Started</h2>

<p>The next to last command triggers a warning because the string
