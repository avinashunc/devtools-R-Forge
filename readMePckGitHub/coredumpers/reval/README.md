    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-reval-formula-evaluator-for-r" class="anchor" href="#reval-formula-evaluator-for-r" aria-hidden="true"><span class="octicon octicon-link"></span></a>reval: formula evaluator for R</h1>

<p><a href="https://travis-ci.org/giupo/reval"><img src="https://camo.githubusercontent.com/4573f180276a432fa3436f0da026fc8859c7e0bb/68747470733a2f2f7472617669732d63692e6f72672f676975706f2f726576616c2e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/giupo/reval.svg?branch=master" style="max-width:100%;"></a></p>

<p>Assume you have a bunch of formulas you need to compute in an orderly
fashion, respecting dependencies expressed by formulas, if you try to
visualize this concept, you can model a computation network, a graph,
where all the formulas express a node with edges towards other formulas
or "sources" (scalars) data not depending on other formulas.</p>

<p>To simplify: you have <code>A</code>, <code>B</code> who represent sources (data obtained from 
other places, data-entry, whatever). <code>C</code> is a piece of data with a formula 
attached</p>

<p><code>C</code> declares its formula (and dependencies).</p>

<p>All (<code>A</code>, <code>B</code>, <code>C</code>) can have metadata subject to indexing to
retrieve data later on. This is totally schema-free.</p>

<p>So, let's render it:</p>

<p><code>A-&gt;C B-&gt;C</code></p>

<p><code>A</code> and <code>B</code> contribute to <code>C</code>. This inductively can realize a
huge network, so there are methods to \link{move} inside the network, to inspect
search data (in some cases it would be useful to know who contributes to what
and viceversa).</p>

<p>Let's init an <code>DAG</code>:</p>

<p><code>dag &lt;- DAG()</code></p>

<p>Now add some sources</p>

<p><code>dag["A"] &lt;- valueA</code>
<code>dag["B"] &lt;- valueB</code></p>

<p><code>valueA</code> and <code>valueB</code> can be anything R is able to handle.</p>

<p>Now let's add a node with a formula. Define a function:</p>

<p><code>functionC &lt;- function(A,B) {
    A+B
}</code></p>

<p>This function sums <code>A</code> and <code>B</code> (it's <em>IMPORTANT</em> that function
 args have same identifiers of nodes we have just added to the <code>ETL</code>)</p>

<p><code>dag["C"] &lt;- functionC</code></p>

<p>Of course it could have been written as:</p>

<p><code>dag["C"] &lt;- function(A,B) { A+B }</code></p>

<p>The <code>DAG</code> class will identify the function args, and define the dependencies to build
 it's internal network for later processing.</p>
