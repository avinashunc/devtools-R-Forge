    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-rscribd-package" class="anchor" href="#rscribd-package" aria-hidden="true"><span class="octicon octicon-link"></span></a>rscribd Package</h1>

<p><strong>rscribd</strong> allows users to publish and retrieve documents from the <a href="http://www.scribd.com/">Scribd</a> publishing platform.</p>

<h2>
<a id="user-content-requirements-and-installation" class="anchor" href="#requirements-and-installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Requirements and Installation</h2>

<p><a href="https://travis-ci.org/leeper/rscribd"><img src="https://camo.githubusercontent.com/ae64b11f0d289c7dc701d917326ebdcee25c1c9e/68747470733a2f2f7472617669732d63692e6f72672f6c65657065722f727363726962642e706e673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/leeper/rscribd.png?branch=master" style="max-width:100%;"></a></p>

<p><strong>rscribd</strong> will eventually be available on <a href="http://cran.r-project.org/web/packages/rscribd/index.html">CRAN</a>, so that it can be installed using:</p>

<pre><code>install.packages('rscribd')
</code></pre>

<p>The development version can be installed directly from GitHub using <code>devtools</code>:</p>

<pre><code>if(!require('devtools')) {
    install.packages('devtools')
    library('devtools')
}
install_github('leeper/rscribd')
</code></pre>

<h2>
<a id="user-content-example-usage" class="anchor" href="#example-usage" aria-hidden="true"><span class="octicon octicon-link"></span></a>Example Usage</h2>

<p>Below are some simple code examples demonstrating package functionality.</p>

<p><a href="http://www.scribd.com/developers/platform#limits">Usage limits</a> for the API are very generous and it is highly unlikely you will reach them.</p>

<h3>
<a id="user-content-authentication-options" class="anchor" href="#authentication-options" aria-hidden="true"><span class="octicon octicon-link"></span></a>Authentication options</h3>

<p>The Scribd API has several possible ways to authenticate requests. The simplest involves passing an API key to any <strong>rscribd</strong> function using the <code>api_key</code> argument. If this value is not specified, <strong>rscribd</strong> will look for an API key in <code>options(scribd_api_key)</code>. If none is provided, any operation will fail.</p>

<p>In addition to API key authentication, API requests can be further authenticated via request signing (essentially MD5 hashing), which provides an added layer of security that prevents others from using an exposed API key for unauthorized use. To sign requests, your API account must be configured in the <a href="http://www.scribd.com/account-settings/api">Scribd API options</a> to have "Require API Signature" set to "Require signature". Then, in addition to the <code>api_key</code> argument, all function calls must additionally include the API Secret Key as the <code>secret_key</code> argument (or have a secret key set globally with <code>options(scribd_secret_key)</code>). All functions will work without a secret key, by defaulting to basic authentication (described above).</p>

<p>If you are trying to build a public-facing app on top of <strong>rscribd</strong>, you may also want to leverage a user-specific session key returned by <code>scribd_login</code>. This allows user-specific operations without requiring users to register an API key. Once a session key is obtained, it can be passed to any <strong>rscribd</strong> function using the <code>session_key</code> argument (or have that argument set globally using <code>options(scribd_session_key)</code>). The session key is reasonably long lived. If this value is not specified, all operations are performed (by default) on the account associated with the API key.</p>

<h3>
<a id="user-content-uploading-documents-to-scribd" class="anchor" href="#uploading-documents-to-scribd" aria-hidden="true"><span class="octicon octicon-link"></span></a>Uploading documents to Scribd</h3>

<p>Documents can be uploaded to Scribd in two ways: from a local file or via URL. Supported file types are: pdf, txt, ps, rtf, epub, key, odt, odp, ods, odg, odf, sxw, sxc, sxi, sxd, doc, ppt, pps, xls, docx, pptx, ppsx, xlsx. Image files formats are not allowed.</p>

<p>To upload a document, you can simply use:</p>

<pre><code>mydoc &lt;- doc_upload("myfile.pdf")
</code></pre>

<p>An optional <code>wait</code> argument to <code>doc_upload</code> waits and repeatedly checks to see if the upload was successful before returning to the console. If the upload fails, an error is produced. If <code>wait=FALSE</code>, no checking is performed (but can be performed manually using <code>conversion_status</code>).</p>

<p>Once uploaded, it is possible to view document settings and metadata using <code>doc_settings</code>:</p>

<pre><code>doc_settings(mydoc)
</code></pre>

<p>These settings can be changed using <code>doc_change</code>, for example to modify a title:</p>

<pre><code>doc_change(mydoc, title = "My first rscribd upload")
</code></pre>

<h3>
<a id="user-content-collections" class="anchor" href="#collections" aria-hidden="true"><span class="octicon octicon-link"></span></a>Collections</h3>

<p>Documents can also be organized into "collections," which combine related documents together.</p>

<p>To create a collection, usinge <code>coll_create</code>:</p>

<pre><code>mycol &lt;- coll_create("My rscribd documents", "A collection of rscribd documents")
</code></pre>

<p>You can change the title, description, and access setting for a collection using <code>coll_update</code>.</p>

<p>To add a document to a collection, use <code>coll_add</code>. You can use <code>coll_remove</code> to remove a document:</p>

<pre><code>coll_add(mycol, mydoc)
coll_remove(mycol, mydoc)
</code></pre>

<p><code>coll_list</code> returns a list of documents contained in a collection:</p>

<pre><code>coll_list(mycol)
</code></pre>

<p><code>coll_delete</code> deletes a collection. <code>doc_delete</code> deletes a document from Scribd entirely.</p>
