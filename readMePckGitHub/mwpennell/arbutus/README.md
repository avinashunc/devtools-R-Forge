    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><p><a href="https://travis-ci.org/mwpennell/arbutus"><img src="https://camo.githubusercontent.com/bae66aad5e3ef6662ebe8e235c0284ed2b481d31/68747470733a2f2f7472617669732d63692e6f72672f6d7770656e6e656c6c2f617262757475732e706e673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/mwpennell/arbutus.png?branch=master" style="max-width:100%;"></a></p>

<p><a href="https://github.com/mwpennell/arbutus/raw/master/extra/arbutus_logo.png" target="_blank"><img src="https://github.com/mwpennell/arbutus/raw/master/extra/arbutus_logo.png" alt="arbutus logo" style="max-width:100%;"></a></p>

<p>This package was developed to assess the adequacy of continuous trait models. The package includes functions to parse the output of various model-fitting functions in R, rescale phylogenies according to fitted parameters, simulate datasets, calculate summary statistics and plot results. The methods is described in a forthcoming paper: "Model adequacy and the macroevolution of angiosperm functional traits" <a href="http://biorxiv.org/content/early/2014/04/07/004002">(preprint here)</a>.</p>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>To install the package from github, the easiest way is to install using <a href="https://github.com/hadley/devtools">devtools</a>. Install <code>devtools</code>, then type</p>

<pre><code>library(devtools)
install_github("mwpennell/arbutus")
</code></pre>

<p>The package depends on the <code>ape</code> library</p>

<h2>
<a id="user-content-supported-packages" class="anchor" href="#supported-packages" aria-hidden="true"><span class="octicon octicon-link"></span></a>Supported packages</h2>

<p>The idea behind <code>arbutus</code> is that models are fit to comparative data using other packages and <code>arbutus</code> is used to assess the adequacy of the fitted model. There are no native functions for fitting models or performing model selection within this package. Currently, arbutus can handle objects of the following classes:</p>

<ul class="task-list">
<li>
<code>gfit</code> objects from <code>fitContinuous</code> in <a href="https://github.com/mwpennell/geiger-v2">geiger</a> (&gt;= 2.0)</li>
<li>
<code>fit.mle</code> objects from <code>find.mle</code> in <a href="https://github.com/richfitz/diversitree">diversitree</a> (&gt;= 0.9.7)</li>
<li>
<code>mcmcsamples</code> objects from <code>mcmc</code> in <a href="https://githbu.com/richfitz/diversitree">diversitree</a> (&gt;= 0.9.7)`</li>
<li>
<code>gls</code> objects from <code>gls</code> in <code>nlme</code>
</li>
<li>
<code>pgls</code> objects from <code>pgls</code> in <code>caper</code>
</li>
<li>
<code>phylolm</code> objects from <code>phylolm</code> in <code>phylolm</code>
</li>
<li>
<code>phylo</code> objects in <code>ape</code> format (does not rescale phylogeny)</li>
<li>
<code>multiPhylo</code> objects in <code>ape</code> format (does not rescale phylogeny)</li>
</ul>

<p>We plan to add more package interfaces as this project progresses.</p>
