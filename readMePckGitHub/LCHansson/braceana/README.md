    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-braceana" class="anchor" href="#braceana" aria-hidden="true"><span class="octicon octicon-link"></span></a>Braceana</h1>

<p>Version 0.1.1</p>

<p>Watch a demo of Braceana in action <a href="http://lchansson.shinyapps.io/braceana">here</a>!</p>

<p>Braceana is an attempt to create a dirt simple publishing platform in R/shiny. It is licensed under the AGPL version 3 (see enclosed LICENSE).</p>

<ul class="task-list">
<li>Are you a web developer, looking for a quick-and-dirty CMS for quick publishing?</li>
<li>Are you looking for lightning fast responsive design, easily managed SEO, or an awesome plugin system based on your your favorite scripting language?</li>
<li>Are you tired of bloated CMSes capable of doing everything, but being good at nothing - especially not <em>your favorite web development thing</em>?</li>
</ul>

<p>Then you should probably look somewhere else.</p>

<h4>
<a id="user-content-designed-for-r-developers-and-the-web" class="anchor" href="#designed-for-r-developers-and-the-web" aria-hidden="true"><span class="octicon octicon-link"></span></a>Designed for R developers and the web</h4>

<p>On the other hand, if you're an R developer, analyst, or statistical programmer who shares the love for easy publishing using Rmarkdown, Shiny widgets and ggvis graphics, then Braceana might actually be for you.</p>

<p>Braceana is a simple platform using Rmarkdown (*.Rmd) as its sole publishing format. It is designed to take care of HTML rendering, theming, styling, and online publishing for you. In a similar fashion to Jekyll, it supports Pages and Blog posts and is easily extendable with Shiny widgets like ggvis graphics or RCharts plots. However, unlike Jekyll, Braceana is not easily configurable and does not support any kinds of plugins. It does, however, come with custom CSS support so you can make it look any way you want to, if you want to.</p>

<p>If you're accustomed to customizing Shiny apps and know some HTML, you can easily customize most parts of Braceana to do exactly whatever you want it to.</p>

<h2>
<a id="user-content-features" class="anchor" href="#features" aria-hidden="true"><span class="octicon octicon-link"></span></a>Features</h2>

<ul class="task-list">
<li>Auto-generate HTML from R Markdown files</li>
<li>Supports pages and blog posts</li>
<li>Run Shiny widgets and full-scale apps</li>
<li>Bootstrap based CSS, with custom styling possible</li>
<li>Full-time hackers familiar with Shiny can customise Braceana to their heart's content</li>
</ul>

<h2>
<a id="user-content-requirements" class="anchor" href="#requirements" aria-hidden="true"><span class="octicon octicon-link"></span></a>Requirements</h2>

<p>Braceana requires R packages <code>shiny</code> (version &gt;=0.10.1), <code>markdown</code> (version &gt;=0.7.4), and <code>rmarkdown</code> (version &gt;=0.3.3).</p>

<p>Braceana can be run locally using the 'shiny' package, or remotely using a Shiny hosting service like <a href="http://shinyapps.io">Shinyapps.io</a>.</p>

<h2>
<a id="user-content-using-braceana" class="anchor" href="#using-braceana" aria-hidden="true"><span class="octicon octicon-link"></span></a>Using Braceana</h2>

<p>Start by cloning the <a href="https://github.com/lchansson/braceana">Braceana repo</a> from Github (or, alternatively, downloading a zip file containing the repo).</p>

<p>Braceana is ready to use on the fly. Make sure you have the proper dependencies installed, and copy the files found in the <code>examples</code> folder to the <code>pages</code>and <code>blog</code> folders found in the root folder. Then fire up RStudio and press Run App, or do <code>setwd("my/path/to/braceana")</code> and run <code>shiny::runApp()</code>.</p>

<h4>
<a id="user-content-deployment" class="anchor" href="#deployment" aria-hidden="true"><span class="octicon octicon-link"></span></a>Deployment</h4>

<p>Braceana supports publishing to <a href="http://shinyapps.io">ShinyApps.io</a>. Simply make sure to follow the instructions for activating your account that can found on their web page, and deploy your app (including all pages and blog posts you might want to deploy) using <code>shinyapps::deployApp()</code>.</p>

<p>For an example, see <a href="http://lchansson.shinyapps.io/braceana">this website</a>.</p>
