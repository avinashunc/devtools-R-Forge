    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h2>
<a id="user-content-port-of-gwidgets-to-the-web" class="anchor" href="#port-of-gwidgets-to-the-web" aria-hidden="true"><span class="octicon octicon-link"></span></a>Port of gWidgets to the web</h2>

<p>This package implements the <code>gWidgets</code> API for web
programming. Deployment is through <code>Rook</code>, best suited for local use
or limited internet usage. For wider deployment, there is
<code>gWidgetsWWW2.rapache</code> which runs under Jeffrey Horner's rapache
module for the apache webserver.</p>

<p>Primarily, the package turns the gWidgets commands into
<code>JavaScript</code> commands. The <code>ExtJS</code> javascript libraries (<a href="http://www.sencha.com">www.sencha.com</a>)
are used for this, as they provide an excellent programming
environment for <code>JavaScript</code>. Of course, the <code>R</code> user need not know any
<code>JavaScript</code>, or even any web technologies -- not even HTML. Basically,
most simple gWidgets scripts will just work.</p>

<p>If installed from github, the extjs files will be downloaded. If
installed as a R package, the commmand <code>download_extjs</code> must be issued
to download the extjs files (67Mb) and isntall them within the package
directory.</p>

<h2>
<a id="user-content-example" class="anchor" href="#example" aria-hidden="true"><span class="octicon octicon-link"></span></a>Example</h2>

<p>To use <code>gWidgetsWWW2</code> locally, one writes scripts to be executed by Rook.
The basic "Hello World" app with a window, container, button and
callback is created in a script <code>hello.R</code> and contains:</p>

<pre><code>    w &lt;- gwindow("Hello world example")
    g &lt;- ggroup(cont=w, width=300, height=300)
    b &lt;- gbutton("click me", cont=g, handler=function(h,...) {
      galert("Hello World", parent=w)
    })
</code></pre>

<p>This script is turned into a web page through the <code>load_app</code> function:</p>

<pre><code>    require(gWidgetsWWW2)
    load_app("hello.R", app_name="hello")
</code></pre>

<p>The app is then mapped to the url</p>

<blockquote>
<pre><code>http://127.0.0.1:PORT/custom/hello/indexgw.rhtml
</code></pre>
</blockquote>

<p>The "PORT" will be that for R's internal help server. The <code>port</code>
argument can pass in a value if that has not yet been started, with a
default of 9000.</p>

<p>The "hello" in the url matches that given by the <code>app_name</code> argument. </p>

<p>Read on for notes on deployment beyond the local computer.</p>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>Installing the package from GitHub is made easy if the <code>devtools</code>
package of H. Wickham is installed:</p>

<pre><code>    library(devtools)
    install_github("gWidgetsWWW2", "jverzani")
</code></pre>

<p>Otherwise, a) install git, b) clone the project c) use +R CMD
INSTALL+, or some such to install from a local set of files.</p>

<h2>
<a id="user-content-graphics" class="anchor" href="#graphics" aria-hidden="true"><span class="octicon octicon-link"></span></a>Graphics</h2>

<p>There is no interactive graphic device, though one can use several
non-interactive graphics devices, in particular: <code>png</code>, through the <code>gimage</code> widget; and <code>RSVGTipsDevice</code>,
through the <code>gsvg</code> widget. The basic use for each is similar.</p>
