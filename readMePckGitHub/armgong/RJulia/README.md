    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-rjulia--rjulia" class="anchor" href="#rjulia--rjulia" aria-hidden="true"><span class="octicon octicon-link"></span></a>RJulia / rjulia</h1>

<p>RJulia (<code>rjulia</code> is the name of the R package, <code>RJulia</code> the github
repository name) provides an interface between R and Julia. It allows a user to run a script in Julia from R, and maps objects between the two languages.</p>

<p>It currently supports use on Linux and Windows from the console, but build on Windows only for advance users.</p>

<h2>
<a id="user-content-install" class="anchor" href="#install" aria-hidden="true"><span class="octicon octicon-link"></span></a>Install</h2>

<ol class="task-list">
<li>
<p>Install julia v0.33 or the current master branch; R version &gt;=3.1.0.</p>

<p><strong>WARNING: due to julia api rapidly change between releases, each time you upgrade or downgrade julia, rjulia need to be recompiled and reinstalled.</strong></p>

<p><strong>WARNING: due to <a href="https://github.com/JuliaLang/julia/issues/10085">https://github.com/JuliaLang/julia/issues/10085</a>, rjulia will crash when run certain julia code on julia 0.35 or 0.3 head branch, so please install julia 0.33 or 0.4-dev</strong></p>

<p><strong>but you can still try julia 0.35 or 0.3 head branch, most demo run ok ,except the testparallel demo .</strong></p>
</li>
<li><p>add /bin to your system PATH variable if needed.</p></li>
<li><p>"Compile" RJulia.</p></li>
<li><p>If you want to be able to use R or Julia objects that contain NA values
or factors or data frames, the Julia packages <code>DataArrays</code> and <code>DataFrames</code> must be installed.</p></li>
<li><p>Windows user please download built binary package from <a href="https://github.com/armgong/rjulia/releases">https://github.com/armgong/rjulia/releases</a>  <strong>(Caution: use master branch build first, nextgen branch build not stable)</strong></p></li>
</ol>

<h2>
<a id="user-content-simple-run-it" class="anchor" href="#simple-run-it" aria-hidden="true"><span class="octicon octicon-link"></span></a>Simple Run it</h2>

<p>library(rjulia)</p>

<p>julia_init() #<strong>(will auto find your julia home)</strong></p>

<p>julia_eval("1+1")</p>

<h2>
<a id="user-content-demo" class="anchor" href="#demo" aria-hidden="true"><span class="octicon octicon-link"></span></a>Demo</h2>

<p>please see the <code>*.R</code> files in the <code>demo/</code> directory, or use</p>

<pre><code>    demo(package = "rjulia")
</code></pre>

<a id="user-content-doc" class="anchor" href="#doc" aria-hidden="true"><span class="octicon octicon-link"></span></a>Doc</h2>

<h2>
<a id="user-content-important-information" class="anchor" href="#important-information" aria-hidden="true"><span class="octicon octicon-link"></span></a><strong>Important Information</strong>
</h2>

<p>We now develop on two branches. The <code>master</code> branch is the stable version,
the <code>nextgen</code> branch is the development version. The <code>nextgen</code> branch uses
the pure julia C API to call a julia function, don't mix julia script and c code,
<strong>so it is faster than the master branch and uses less memory, but it is less stable</strong>.</p>

<p>If you want to use the <code>nextgen</code> branch, please ensure <code>DataFrames</code> (julia) package
version &gt;= 0.60.</p>
</article>
