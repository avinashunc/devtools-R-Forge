    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-trakt-" class="anchor" href="#trakt-" aria-hidden="true"><span class="octicon octicon-link"></span></a>tRakt <a href="https://travis-ci.org/jemus42/tRakt"><img src="https://camo.githubusercontent.com/21a1c847af491a7a88006c86be27fe29fd5d2080/68747470733a2f2f7472617669732d63692e6f72672f6a656d757334322f7452616b742e737667" alt="Build Status" data-canonical-src="https://travis-ci.org/jemus42/tRakt.svg" style="max-width:100%;"></a>
</h1>

<p>This is <code>tRakt</code> version <code>0.13.1</code>.<br>
It contains functions to pull data from <a href="http://trakt.tv/">trakt.tv</a>.</p>

<p>It's an <a href="http://r-project.org">R package</a> primarily used by (i.e. build for) <a href="http://trakt.jemu.name">this webapp</a>, but you can fiddle around with it if you like.<br>
There might be some interesting things to play around with, and I've tried some of them <a href="http://cran.r-project.org/web/packages/tRakt/vignettes/tRakt-Usage.html">here</a> (Also included as a package vignette).</p>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>Get it from <a href="http://cran.r-project.org">CRAN</a>:</p>

<pre><code>install.packages("tRakt")
</code></pre>

<p>Or just to get the latest dev version from GitHub:</p>

<pre><code>if (!require("devtools")) install.packages("devtools")
devtools::install_github("jemus42/tRakt")
library("tRakt")
</code></pre>

<h2>
<a id="user-content-setting-credentials" class="anchor" href="#setting-credentials" aria-hidden="true"><span class="octicon octicon-link"></span></a>Setting credentials</h2>

<p>The APIv2 requires at least a <code>client id</code> for the API calls.<br>
Calling <code>get_trakt_credentials()</code> will set everything up for you, but you either have to
manually plug your values in (see <code>?get_trakt_credentials()</code>), or have a <code>key.json</code> sitting either in the working directory or in <code>~/.config/trakt/key.json</code>.<br>
It should look like this:</p>

<pre><code>{
  "username": "yourusername",
  "client.id": "&lt;APIv2 client id&gt;",
  "client.secret": "&lt;APIv2 client secret&gt;"
}
</code></pre>

<ul class="task-list">
<li>
<code>username</code> Optional. For functions that pull a user's watched shows or stats (<code>trakt.user.*</code>)</li>
<li>
<code>client.id</code> Required. It's used in the HTTP headers for the API calls, which is kind of a biggie.</li>
<li>
<code>client.secret</code> NYI. Is only required for <code>OAuth2</code> methods, and I don't really intend on using those unless I <em>really really</em> have to.<br>
</li>
</ul>

<p>To get your credentials, <a href="http://trakt.tv/oauth/applications">you have to have an (approved) app over at trakt.tv</a>.<br>
Don't worry, it's really easy to set up. Even I did it.</p>

<h3>
<a id="user-content-use-my-apps-clientid-as-a-fallback" class="anchor" href="#use-my-apps-clientid-as-a-fallback" aria-hidden="true"><span class="octicon octicon-link"></span></a>Use my app's client.id as a fallback</h3>

<p>As a convenience for you, and also to make automated testing a little easier,
<code>get_trakt_credentials()</code> automatically sets my <code>client.id</code> as a fallback, so you theoretically
never need to supply your own credentials. However, if you want to actually use this package for
some project, I do not recommend relying on my credentials. That would make me a sad panda.</p>
