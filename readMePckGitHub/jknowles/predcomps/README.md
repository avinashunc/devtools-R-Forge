    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><p><a href="http://www.davidchudzicki.com/predcomps/">Documentation etc.</a></p>

<p>Interested in getting involved? Here are some ways to help:</p>

<ul class="task-list">
<li>Trying out the package in examples, write about the results. Does it help you better understand your complicated models? If not, what should be different? You can add examples to the documentation here via pull requests, or write in your own space.</li>
<li>Clarifying the language in the documentation - what can be made clearer?</li>
<li>Working on any of the "future work" below</li>
</ul>

<h2>
<a id="user-content-future-work" class="anchor" href="#future-work" aria-hidden="true"><span class="octicon octicon-link"></span></a>Future Work</h2>

<p>(Unsure about the priority of the rest, but the top one is definitely most important.)</p>

<h3>
<a id="user-content-explicit-model-for-puv" class="anchor" href="#explicit-model-for-puv" aria-hidden="true"><span class="octicon octicon-link"></span></a>Explicit model for p(u|v)</h3>

<p>Currently <a href="http://0.0.0.0:4000/more-pairs-and-weights.html">the way we assign weights</a> to sample for p(u|v) (roughly as described in the paper) requires a bit of hand-tweaking to work well in individual examples. It's also hard to generalize to categorical inputs. This may be the biggest barrier to widespread adoption.</p>

<p>As an alternative, perhaps we can explicitly build a model for the desired conditional distribution, e.g. maybe by using something like <a href="https://github.com/kapelner/bartMachine">BART</a>.</p>

<p>Todo:</p>

<ul class="task-list">
<li>implement</li>
<li>see how well it works</li>
</ul>

<h3>
<a id="user-content-categorical-inputs" class="anchor" href="#categorical-inputs" aria-hidden="true"><span class="octicon octicon-link"></span></a>Categorical inputs</h3>

<p>Once we've done "Explicit model for p(u|v)", allowing categorical inputs should be much easier, but there's still some thought required.</p>

<h3>
<a id="user-content-sensivity-analysis" class="anchor" href="#sensivity-analysis" aria-hidden="true"><span class="octicon octicon-link"></span></a>Sensivity Analysis</h3>

<p>Some of the examples show how to do sensitivity analysis in the spirit of this package, but it'd be great to have that do it for you.</p>

<h3>
<a id="user-content-variable-importance" class="anchor" href="#variable-importance" aria-hidden="true"><span class="octicon octicon-link"></span></a>"Variable Importance"</h3>
