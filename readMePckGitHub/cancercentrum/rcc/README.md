    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-rcc" class="anchor" href="#rcc" aria-hidden="true"><span class="octicon octicon-link"></span></a>rcc</h1>

<p>Various R-functions for daily work at the Swedish Regional Cancer Centers.</p>

<p>The package is currently beta and should be used with some caution!</p>

<p>You are very very welcome to try it out! Copy the folowing to your R console to install the package (why not <code>devtools::install_github("cancercentrum/rcc")</code>? See: <a href="https://github.com/cancercentrum/rcc/wiki">https://github.com/cancercentrum/rcc/wiki</a>).</p>

<div class="highlight highlight-r"><pre><span class="pl-c"># Download and install</span>
source(<span class="pl-s"><span class="pl-pds">"</span>http://cancercentrum.se/Global/OCVast/publikationer/statistik/install_rcc.txt<span class="pl-pds">"</span></span>)

<span class="pl-c"># The package is installed and can be loaded:</span>
library(<span class="pl-smi">rcc</span>)

<span class="pl-c"># It is always difficult to use a package without knowledge of its functionalities. </span>
<span class="pl-c"># Start by reading the package documentation:</span>
?<span class="pl-smi">rcc</span></pre></div>

<h1>
<a id="user-content-contact-questions-bug-reports-etc" class="anchor" href="#contact-questions-bug-reports-etc" aria-hidden="true"><span class="octicon octicon-link"></span></a>Contact (questions, bug reports etc)</h1>

<p>If you have any questions about anything at all ... or if you find bugs, unclear or missleading documentation (might very well be the case, 
it would not surprise me if I for example have forgotten to update some piece of documentation even though the documented function itself 
has been updated or changed), please let me know!</p>

<p>I would be very happy if your first attempt of contact would be trough the Github issues-functionality:
<a href="https://github.com/cancercentrum/rcc/issues">https://github.com/cancercentrum/rcc/issues</a>
If you are not able to do that (might require that you first register a Github account etc), please let me know!</p>

<h1>
<a id="user-content-about-operating-system" class="anchor" href="#about-operating-system" aria-hidden="true"><span class="octicon octicon-link"></span></a>About Operating System</h1>

<p>I have developed the package on my Mac computer. The Mac runs on a Unix-version called Darwin. There are sometimes important differences compared to Windows machines.
If you use Windows, you might corrently explore some problems regarding:</p>

<ul class="task-list">
<li>Encoding for non ASCII-letters (basically å/Å, ä/Ä and ö/Ö). This might couse formatting errors when using the decode/code-functions. It is mostly a problem of aestetics ... 
but still anoying ...</li>
<li>The "get_document"-function (accessing INCA documentation such as variable specifications, inclussion criterias etcetera) might have problems downloading the files to your computer. It will almost for sure also have problems opening the possibly downloaded documents.</li>
<li>There is an underlying difference in how dates (especially week numbers) are treated in Unix and Windows. I have tried to avoid this problem in the as.Dates/rcc.Dates-function but if you find something strange, please let me know!</li>
</ul>

<p>My vission is to get a stable package version working for Windows as well as Unix (Mac) but all help regarding this is more than welcome!</p>
