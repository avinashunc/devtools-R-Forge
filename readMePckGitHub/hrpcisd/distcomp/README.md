<p>Then, you will find a document that describes several examples
installed under the R library tree. For example:</p>

<div class="highlight highlight-r"><pre>list.files(system.file(<span class="pl-s"><span class="pl-pds">"</span>doc<span class="pl-pds">"</span></span>, <span class="pl-v">package</span><span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>distcomp<span class="pl-pds">"</span></span>))
list.files(system.file(<span class="pl-s"><span class="pl-pds">"</span>doc_src<span class="pl-pds">"</span></span>, <span class="pl-v">package</span><span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>distcomp<span class="pl-pds">"</span></span>))</pre></div>

<p>The examples described in the paper are available as follows:</p>

<div class="highlight highlight-r"><pre>list.files(system.file(<span class="pl-s"><span class="pl-pds">"</span>ex<span class="pl-pds">"</span></span>, <span class="pl-v">package</span><span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>distcomp<span class="pl-pds">"</span></span>))</pre></div>

<p>Use of this package requires some configuration. In particular, to run
the examples on a local machine where a single <code>opencpu</code> server will
be emulating several sites, a suitable R profile needs to be set
up. That profile will be something along the lines of</p>

<pre lang="r,"><code>library(distcomp)
distcompSetup(workspace="full_path_to_workspace_directory",
              ssl.verifyhost=FALSE, ssl.verifypeer=FALSE)
</code></pre>

<p>where the workspace is a directory that the <code>opencpu</code> server can
serialize objects to. On Unix or Mac, the above can be inserted into
an <code>.Rprofile</code> file, but on Windows, we find that the <code>Rprofile.site</code>
file needs to contain the above lines.</p>

<p>The effect of this is that <em>every R</em> process (including the <code>opencpu</code>
process) has access to the <code>distcomp</code> library and the workspace.</p>

<h2>
<a id="user-content-osx-yosemite-update" class="anchor" href="#osx-yosemite-update" aria-hidden="true"><span class="octicon octicon-link"></span></a>OSX Yosemite Update</h2>

<p>We find that on Yosemite (MacOS 10.10.2 for example) there are issues
with the <code>opencpu</code> package. Our suspicion is that this has more to do
with Yosemite than <code>opencpu</code> or <code>httpuv</code>---see the number of DNS
resolver issues people are having with the move to <code>discoveryd</code> via a
google search.</p>

<p>Furthermore, even if <code>opencpu</code> comes up, which it does sometimes after
many tries, you are better off using the ip address <code>127.0.0.1</code> in the
url in place of <code>localhost</code> due to those DNS problems.</p>
