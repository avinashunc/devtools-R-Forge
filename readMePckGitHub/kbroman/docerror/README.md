    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h2>
<a id="user-content-test-roxygen2-documentation-of-data-sets" class="anchor" href="#test-roxygen2-documentation-of-data-sets" aria-hidden="true"><span class="octicon octicon-link"></span></a>test Roxygen2 documentation of data sets</h2>

<p>In documenting datasets with Roxygen2, you don't need to use the
<code>@name</code> directive but can use the name of the dataset as a string
after the Roxygen2 comments, like this:</p>

<pre><code>#' test dataset
#'
#' a dataset for testing use of Roxygen2 to document datasets
#'
#' @docType data
#'
#' @usage data(testdata)
#'
#' @format Just the number \code{0}.
#'
#' @keywords datasets
#'
#' @examples
#' data(testdata)
"testdata"
</code></pre>

<p><em>But</em>, this seems to require <code>LazyData: true</code> in the <code>DESCRIPTION</code>
file.</p>

<h3>
<a id="user-content-branch-master" class="anchor" href="#branch-master" aria-hidden="true"><span class="octicon octicon-link"></span></a>Branch <a href="https://github.com/kbroman/docerror/tree/master">master</a>
</h3>

<ul class="task-list">
<li>The dataset is documented as above</li>
<li>I <em>don't</em> include <code>LazyData: true</code> in the <code>DESCRIPTION</code> file.</li>
</ul>

<p>When I run</p>

<pre><code>R -e 'library(devtools);document()'
</code></pre>

<p>I get the following error:</p>

<pre><code>&gt; library(devtools);document()
Updating docerror documentation
Loading docerror
Error: Failure in roxygen block beginning testdata-data.R:1
object 'testdata' not found
Execution halted
make: *** [doc] Error 1
</code></pre>
