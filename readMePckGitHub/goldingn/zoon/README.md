    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-zoon" class="anchor" href="#zoon" aria-hidden="true"><span class="octicon octicon-link"></span></a>Zoon</h1>

<p>Zoon is a package for the reproducible and shareable analysis of species distribution models with a focus on the ability to compare between models and diagnostic output of models.</p>

<p>An overview of the project can be found <a href="http://www.2020science.net/research/species-distribution-modelling">here</a>.
There is a blog to keep collaborators up to date with progress. This can be found <a href="http://zoonproject.wordpress.com">here</a></p>

<p>Zoon is still being developed. Feel free to clone and use the code, open issues, let us know what you want etc. But don't expect much functionality from the package yet. If you would like to add functionality, please start writing modules!</p>

<h3>
<a id="user-content-to-install-current-development-version" class="anchor" href="#to-install-current-development-version" aria-hidden="true"><span class="octicon octicon-link"></span></a>To install current development version.</h3>

<h3>
<a id="user-content-contributing-modules" class="anchor" href="#contributing-modules" aria-hidden="true"><span class="octicon octicon-link"></span></a>Contributing modules</h3>

<p>Zoon has a modular structure, and we are hoping for user submitted modules. This allows Zoon to keep up to date with the fast-moving SDM field in a way a package maintained by a small team of developers can't. Modules are simple R scripts containing a single function and some metadata. They are currently kept <a href="https://github.com/zoonproject/modules">here</a>. The inputs and outputs of each module type are controlled. A brief description can be found <a href="https://github.com/zoonproject/zoon/blob/master/vignettes/Module_IO_for_devs.Rmd">here</a>. The function <code>BuildModule</code> is used to turn a function in an R session into a module. </p>

<p>Please note, Zoon is still being developed. We would love you to contribute modules, but can't yet guarantee that there won't be major changes that might break modules. We will try to fix user submitted modules if we break them. </p>

<a id="user-content-notes-for-collaborators" class="anchor" href="#notes-for-collaborators" aria-hidden="true"><span class="octicon octicon-link"></span></a>Notes for collaborators</h3>

<p>We are using the <a href="https://google-styleguide.googlecode.com/svn/trunk/Rguide.xml">Google style guide</a> with the exception that function description goes before the function name, not inside the function definition. We are using <a href="http://cran.r-project.org/web/packages/roxygen2/vignettes/roxygen2.html">roxygen2</a> to document the package. Try to keep function names as verbs.</p>
