    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-printr" class="anchor" href="#printr" aria-hidden="true"><span class="octicon octicon-link"></span></a>printr</h1>

<p><a href="https://travis-ci.org/yihui/printr"><img src="https://camo.githubusercontent.com/f80dd67ebdfcdc977301e7f5993d1db20eb1e8be/68747470733a2f2f7472617669732d63692e6f72672f79696875692f7072696e74722e737667" alt="Build Status" data-canonical-src="https://travis-ci.org/yihui/printr.svg" style="max-width:100%;"></a></p>

<p>This is a companion package to <a href="http://yihui.name/knitr"><strong>knitr</strong></a>. Its main
purpose is to extend the S3 generic function <code>knit_print()</code> in <strong>knitr</strong>, which
is the default value of the chunk option <code>render</code>, as explained in the vignette
<a href="http://cran.rstudio.com/web/packages/knitr/vignettes/knit_print.html">knit_print.html</a>.</p>

<p>It has not been released to CRAN yet, and you can install the development
version from my personal repo for now:</p>

<div class="highlight highlight-r"><pre>install.packages(
  <span class="pl-s"><span class="pl-pds">'</span>printr<span class="pl-pds">'</span></span>,
  <span class="pl-v">type</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>source<span class="pl-pds">'</span></span>,

)</pre></div>

<p>To enable the printing methods defined in this package, just <code>library(printr)</code>
in a code chunk (in the beginning) of your <strong>knitr</strong> document. Then some objects
will be printed differently with what you would have seen in a normal R console.
For example:</p>

<ul class="task-list">
<li>matrices, data frames, and contingency tables are printed as tables (LaTeX,
HTML, or Markdown, depending on your output format)</li>
<li>the help page (from <code>?foo</code> or <code>help(foo)</code>) can be rendered as HTML, LaTeX, or
plain text, and you can also specify which section(s) of the help page to
include in the output</li>
<li>the results from <code>browseVignettes()</code>, <code>help.search()</code>, <code>data()</code>, and
<code>vignette()</code> are rendered as tables</li>
<li>the package information from <code>library(help = 'foo')</code> is rendered as plain text</li>
</ul>

<p>For more information, please check out the <a href="http://yihui.name/printr">package
vignette</a>:</p>

<div class="highlight highlight-r"><pre>vignette(<span class="pl-s"><span class="pl-pds">'</span>printr<span class="pl-pds">'</span></span>, <span class="pl-v">package</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>printr<span class="pl-pds">'</span></span>)</pre></div>

<p>You are welcome to contribute more S3 methods to this package, and you may want
to read the existing methods in this package before you get started. In most
cases, I guess you will end up rendering either plain text (see
<code>knit_print.packageInfo</code> for example) or tables (see <code>knit_print.matrix</code>) in the
output.</p>
