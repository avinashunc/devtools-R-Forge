    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-rkoans" class="anchor" href="#rkoans" aria-hidden="true"><span class="octicon octicon-link"></span></a>RKoans</h1>

<p>Rkoans is an interactive tutorial for learning R in a Unit-testing framework.  No previous knowledge of R is required.</p>

<p>As well as being a fun way to learn R, it is also a good way to get an idea of test-driven development.</p>

<h2>
<a id="user-content-getting-started" class="anchor" href="#getting-started" aria-hidden="true"><span class="octicon octicon-link"></span></a>Getting started</h2>

<p>First, you need to clone the Rkoans repo to somewhere you have write access and then change into the Rkoans directory:</p>

<pre><code>git clone https://github.com/DASpringate/Rkoans.git
cd Rkoans
</code></pre>

<p>Then, either fire up R in this directory or start RStudio and create a new project from the Rkoans directory. Then you can load Rkoans package using <code>devtools</code>:</p>

<div class="highlight highlight-r"><pre><span class="pl-c"># Make sure you are in the Rkoans directory</span>
require(<span class="pl-smi">devtools</span>)
load_all(<span class="pl-s"><span class="pl-pds">"</span>.<span class="pl-pds">"</span></span>)</pre></div>

<p><em>Don't install Rkoans as a package because you will need access to some of the folders within as you work through the koans.</em></p>

<h2>
<a id="user-content-studying-the-koans" class="anchor" href="#studying-the-koans" aria-hidden="true"><span class="octicon octicon-link"></span></a>Studying the koans</h2>

<p>To start learning R using the Rkoans, first load Rkoans (see above), then enter:</p>

<div class="highlight highlight-r"><pre>study_koans()</pre></div>

<p>and you should see something similar to:</p>

<pre><code>...
f. Failure: What is true, is true ------------------------------------------------------------------------------------
TRUE == 0 isn't true

g. Failure: What is true, is true ------------------------------------------------------------------------------------
TRUE == 2 isn't true
what_is_true.R is damaging your karma.
Study this koan to gain new insight...
</code></pre>

<p>This is telling you that you have failing tests in the file what_is_true.R.  The koans are all found in the <code>koans</code> directory within Rkoans.  Go to that file in your text editor or RStudio and start editing the tests in order to make them pass.  You will need to fill in the blanks (<code>_</code>) so that the test statement is true.  When you save the file, the koan reader will update your progress.  If all the tests pass on your file, you will then move on to the next one.
If an answer is not immediately evident, drop out of the koan reader (escape in RStudio or C-C if you are running R from the console) and test the expressions or look for help on the R console.</p>

<p>Remember, to ask for help in R, enter <code>?</code> and the function you want to know about:</p>

<div class="highlight highlight-r"><pre><span class="pl-c"># e.g.</span>
`?`(<span class="pl-smi">sign</span>)</pre></div>

<h2>
<a id="user-content-unit-testing-with-testthat" class="anchor" href="#unit-testing-with-testthat" aria-hidden="true"><span class="octicon octicon-link"></span></a>Unit testing with testthat</h2>

<p>Rkoans uses the <a href="https://github.com/hadley/testthat">testthat</a> unit testing framework.  Tests are built from expectations which read almost like English:</p>

<div class="highlight highlight-r"><pre><span class="pl-c"># testthat expectation example.</span>
expect_that(<span class="pl-c1">1</span> <span class="pl-k">+</span> <span class="pl-c1">1</span>, equals(<span class="pl-c1">2</span>))</pre></div>

<p>For a test to pass, the expectation must be met.</p>

<p>For more on using the testthat framework, see this <a href="http://journal.r-project.org/archive/2011-1/RJournal_2011-1_Wickham.pdf">paper</a>.</p>
