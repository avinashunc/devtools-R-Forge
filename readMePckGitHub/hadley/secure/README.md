    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-secure" class="anchor" href="#secure" aria-hidden="true"><span class="octicon octicon-link"></span></a>secure</h1>

<p><a href="https://travis-ci.org/hadley/secure"><img src="https://camo.githubusercontent.com/a8a0deb750e97ba31a7ab5d63055e8093ee8978f/68747470733a2f2f7472617669732d63692e6f72672f6861646c65792f7365637572652e706e673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/hadley/secure.png?branch=master" style="max-width:100%;"></a></p>

<p>The secure package provides a secure vault within a publicly available code repository. It allows you to store private information in a public repository so that only select people can read it. This is particularly useful for testing because you can now store private credentials in your public repo, without them being readable by the world.</p>

<p>Secure is built on top of asymmetric (public/private key) encryption. Secure generates a random master key and uses that to encrypt (with AES256) each file in <code>vault/</code>. The master key is not stored unencrypted anywhere; instead, an encrypted copy is stored for each user, using their own public key. Each user can than decrypt the encrypted master key using their private key, then use that to decrypt each file.</p>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>Secure is currently only available on github. Install it with:</p>

<div class="highlight highlight-R"><pre><span class="pl-c"># install.packages("devtools")</span>
<span class="pl-e">devtools</span><span class="pl-k">::</span>install_github(<span class="pl-s"><span class="pl-pds">"</span>s-u/PKI<span class="pl-pds">"</span></span>) <span class="pl-c"># needed for bug fixes not currently on CRAN</span>
<span class="pl-e">devtools</span><span class="pl-k">::</span>install_github(<span class="pl-s"><span class="pl-pds">"</span>hadley/secure<span class="pl-pds">"</span></span>)</pre></div>

<h2>
<a id="user-content-first-steps" class="anchor" href="#first-steps" aria-hidden="true"><span class="octicon octicon-link"></span></a>First steps</h2>

<p>To get started:</p>

<ul class="task-list">
<li><p>Create a <code>vault</code> directory.</p></li>
<li><p>Add yourself as as user with <code>secure::add_user("your name", local_key())</code>. 
This will add your name and public key to <code>vault/users.json</code>.
(You can add other people from their <code>github_key()</code>s).</p></li>
<li><p>Securely store data: 
<code>secure::encrypt("google", key = "abcdasdf", secret = "asdfsad")</code>.
This creates <code>secure/google.rds.enc</code>, an encrypted rds file.</p></li>
<li><p>Retrieve encrypted data: <code>secure::decrypt("google")</code>. This decrypts
the encrypted file using your private key.</p></li>
</ul>

<h2>
<a id="user-content-in-a-package" class="anchor" href="#in-a-package" aria-hidden="true"><span class="octicon octicon-link"></span></a>In a package</h2>

<ul class="task-list">
<li><p>Create <code>inst/vault</code> and add <code>secure</code> to the <code>Suggests</code> field in the 
<code>DESCRIPTION</code> (or run <code>secure::use_secure()</code>).</p></li>
<li><p>If you use travis, add the public key for your travis repo:
<code>secure::add_user("travis", travis_key("user/repo"))</code>.</p></li>
<li><p>When developing locally, you can use all functions as is. They look for
a vault in the working directory.</p></li>
<li>
<p>In tests, supply the package name to the <code>vault</code> argument. For example,
one of the tests for the secure package looks like this:</p>

<div class="highlight highlight-R"><pre>  test_that(<span class="pl-s"><span class="pl-pds">"</span>can decrypt secrets<span class="pl-pds">"</span></span>, {
    <span class="pl-c"># Skips the test if doesn't have the key to open the secure vault</span>
    skip_when_missing_key(<span class="pl-s"><span class="pl-pds">"</span>secure<span class="pl-pds">"</span></span>)

    <span class="pl-c"># Decrypt a file stored in secure/inst/vault</span>
    <span class="pl-smi">test</span> <span class="pl-k">&lt;-</span> decrypt(<span class="pl-s"><span class="pl-pds">"</span>test<span class="pl-pds">"</span></span>, <span class="pl-v">vault</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>secure<span class="pl-pds">"</span></span>)
    expect_equal(<span class="pl-smi">test</span><span class="pl-k">$</span><span class="pl-smi">a</span>, <span class="pl-c1">1</span>)
    expect_equal(<span class="pl-smi">test</span><span class="pl-k">$</span><span class="pl-smi">b</span>, <span class="pl-c1">2</span>)
  })</pre></div>
</li>
</ul>
