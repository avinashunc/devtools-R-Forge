    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-gg2v" class="anchor" href="#gg2v" aria-hidden="true"><span class="octicon octicon-link"></span></a>gg2v</h1>

<p>A quick and dirty experiment exploring rendering ggplot2 graphics using vega.  </p>

<p>This approach uses relatively little existing ggplot2 code, and the semantics may be slightly different. It is likely to converge over time as both vega and gg2v become more mature, but there's no expectation that ggplot2 and gg2v will ever be exactly concordances.  It's probably easier to think of gg2v as an alternative specification and rendering of the graphics DSL defined by ggplot2.</p>

<p>Please note that this is an <strong>experimental prototype</strong>, and is unlikely to be maintained long-term (unless someone else volunteers). </p>

<h1>
<a id="user-content-currently-supported" class="anchor" href="#currently-supported" aria-hidden="true"><span class="octicon octicon-link"></span></a>Currently supported</h1>

<ul class="task-list">
<li>scales: 

<ul class="task-list">
<li>x and y position scales (continuous and categorical)</li>
<li>colour gradients</li>
<li>limited support for categorical colours</li>
</ul>
</li>
<li>geoms: point, line, path, area, text</li>
<li>stats: none</li>
<li>facets: none</li>
<li>position adjustments: none</li>
</ul>
</article>
