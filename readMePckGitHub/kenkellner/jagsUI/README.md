    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-jagsui" class="anchor" href="#jagsui" aria-hidden="true"><span class="octicon octicon-link"></span></a>jagsUI</h1>

<p>This package runs JAGS (Just Another Gibbs Sampler) analyses from within R. It acts as a wrapper and alternative interface for the functions in the rjags package and adds some custom output and graphical options. </p>

<p>There are several other similar packages (R2jags, runjags, and of course rjags itself). I wrote this one to allow tighter control over the JAGS adaptive and burn-in periods (see the issue described here: <a href="http://stats.stackexchange.com/questions/45193/r2jags-does-not-remove-the-burn-in-part-sometimes">http://stats.stackexchange.com/questions/45193/r2jags-does-not-remove-the-burn-in-part-sometimes</a>) and to simplify other common tasks such as formatting data, plotting results, and running chains in parallel.</p>

<p>To install the package independent of CRAN:</p>

<ol class="task-list">
<li>Install the latest version of JAGS on your computer (<a href="http://sourceforge.net/projects/mcmc-jags/files/JAGS/">http://sourceforge.net/projects/mcmc-jags/files/JAGS/</a>). If you have a 64-bit OS, you may need to install both the 32 and 64 bit versions of JAGS (if available).</li>
<li>Download the latest source package .tar.gz or Windows binary .zip file from the 'Release' tab of the repository (<a href="https://github.com/kenkellner/jagsUI">https://github.com/kenkellner/jagsUI</a>). </li>
<li>
<p>The jagsUI library requires packages rjags and coda. If you have previously installed the coda and/or rjags packages, I highly recommend removing and re-installing them first:</p>

<pre><code>remove.packages(c('coda','rjags'))
install.packages(c('coda','rjags'))
</code></pre>
</li>
<li>Finally, in R or Rstudio, choose to install a package from a local/archive file and select the downloaded package.</li>
</ol>

<p>Alternatively, you can install directly from R, using package devtools (which installs the other required packages automatically). Devtools may complain that Rtools are not installed, but they are not necessary to install this package, so you can safely ignore the warning.</p>

<pre><code>remove.packages(c('coda','rjags'))
library(devtools)
install_github("kenkellner/jagsUI")
</code></pre>

<p>Acknowledgments: Martyn Plummer, developer of the excellent JAGS software package and the rjags R package;  Andrew Gelman, Sibylle Sturtz, Uwe Ligges, Yu-Sung Su, and Masanao Yajima, developers of the R2WinBUGS and R2jags packages on which my work is based; Robert Swihart, Marc Kery, Jerome Guelat, and Michael Schaub, who tested and provided helpful suggestions for early versions of the package.</p>
