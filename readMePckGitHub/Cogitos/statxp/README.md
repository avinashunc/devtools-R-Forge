    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-statxp" class="anchor" href="#statxp" aria-hidden="true"><span class="octicon octicon-link"></span></a>STATXP</h1>

<ul class="task-list">
<li>
<strong>Author:</strong> Guillaume T. Vallet, <a href="mailto:gtvallet@gmail.com">gtvallet@gmail.com</a>, Université de Montréal, CRIUGM</li>
<li>
<strong>Version:</strong> 0.1</li>
<li>
<strong>Date:</strong> 2014/05/08</li>
</ul>

<p><em>Statxp</em> is R package developed for a personal use to improve some experimental analysis processing. 
<em>Statxp</em> offers a set of functions to clean data, export and format results into a <a href="http://yihui.name/knitr/">knitr</a>/<a href="http://www.stat.uni-muenchen.de/%7Eleisch/Sweave/">sweave</a> document.</p>

<p>The detailed description of the functions as well as examples are provided in the R documentation. 
Type <code>?*function_name*</code> in your R console to access it.</p>

<a id="user-content-dependencies" class="anchor" href="#dependencies" aria-hidden="true"><span class="octicon octicon-link"></span></a>
<a name="user-content-dependencies"></a>Dependencies</h2>

<p><em>Predata</em> depends on the <code>dplyr</code> package. 
You can install <em>dplyr</em> by typing <code>install.packages('dplyr')</code> in your R console.</p>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>
<a name="user-content-install"></a>Installation</h2>

<p>To install a R package from Github, you first need to install the devtools package.
In R, type <code>install.packages('devtools')</code>. 
Then install <em>statxp</em> with the following command : <code>install_github('Cogitos/statxp')</code>.
And now enjoy the package!</p>

<h2>
<a id="user-content-functions" class="anchor" href="#functions" aria-hidden="true"><span class="octicon octicon-link"></span></a>
<a name="user-content-functions"></a>Functions</h2>

<h3>
<a id="user-content-filtrt--outliers" class="anchor" href="#filtrt--outliers" aria-hidden="true"><span class="octicon octicon-link"></span></a>filtRT &amp; outliers</h3>

<p>These two functions are designed to filter reaction times (or other dependent variables) by thresholds (minimum and/or maximum values) and/or standard deviations from the mean. 
These functions return the cleaned data frame as well as how many data were exclude by kind of filter.</p>

<p>The difference between <em>filtRT</em> and <em>outliers</em> is that filtRT uses outliers to filter the data base on different conditions (e.g. by subject and independent variables).</p>

<h3>
<a id="user-content-faov-ftt--fmsd" class="anchor" href="#faov-ftt--fmsd" aria-hidden="true"><span class="octicon octicon-link"></span></a>faov, ftt &amp; fmsd</h3>

<p>These functions allow to easily format for knitr/sweave results from an ANOVA (faov, computed with the <a href="http://cran.r-project.org/web/packages/ez/index.html">ez</a> package), a <em>t</em> test of Student (ftt) and simple a mean with its related standard deviation (fmsd).</p>

<p>The formatting is done according to the <a href="http://www.apastyle.org/">APA</a> guidelines.
Different options allow to add the<em>d</em> of Cohen, change the format, etc.</p>

<h3>
<a id="user-content-writeaov" class="anchor" href="#writeaov" aria-hidden="true"><span class="octicon octicon-link"></span></a>writeAOV</h3>

<p>This function was designed to easily export an ANOVA result (from the ez package) into a csv file.
The options allow to add descriptive statistics and/or posthoc analysis.</p>

<h3>
<a id="user-content-figtheme" class="anchor" href="#figtheme" aria-hidden="true"><span class="octicon octicon-link"></span></a>figtheme</h3>

<p>This function return a ggplot graphic which better suits for publication with a white theme, bigger fonts, and a choice of brewer color set. </p>
