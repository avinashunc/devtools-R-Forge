    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-feddata" class="anchor" href="#feddata" aria-hidden="true"><span class="octicon octicon-link"></span></a>FedData</h1>

<p>An <em>R</em> package implementing functions to automate downloading geospatial data available from several federated data sources (mainly sources maintained by the US Federal government). Currently, the package allows for retrieval of five datasets: </p>

<ul class="task-list">
<li>The <a href="http://ned.usgs.gov">National Elevation Dataset (NED)</a> digital elevation models (1 and 1/3 arc-second; USGS)</li>
<li>The <a href="http://nhd.usgs.gov">National Hydrography Dataset (NHD)</a> (USGS)</li>
<li>The <a href="http://websoilsurvey.sc.egov.usda.gov/">Soil Survey Geographic (SSURGO) database</a> from the National Cooperative Soil Survey (NCSS), which is led by the Natural Resources Conservation Service (NRCS) under the USDA,</li>
<li>The <a href="http://www.ncdc.noaa.gov/data-access/land-based-station-data/land-based-datasets/global-historical-climatology-network-ghcn">Global Historical Climatology Network (GHCN)</a>, coordinated by National Climatic Data Center at NOAA, and</li>
<li>The <a href="http://www.ncdc.noaa.gov/data-access/paleoclimatology-data/datasets/tree-ring">International Tree Ring Data Bank (ITRDB)</a>, coordinated by National Climatic Data Center at NOAA.</li>
</ul>

<p>Additional data sources are in the works, including global DEM resources (<a href="https://www.ngdc.noaa.gov/mgg/global/global.html">ETOPO1</a>, <a href="http://www2.jpl.nasa.gov/srtm/">STRM</a>), global soils (<a href="http://webarchive.iiasa.ac.at/Research/LUC/External-World-soil-database/HTML/">HWSD</a>), <a href="http://modis.gsfc.nasa.gov">MODIS</a> satellite data products, the <a href="http://nationalmap.gov/small_scale/">National Atlas</a> (US only), <a href="http://www.naturalearthdata.com">Natural Earth</a>, <a href="http://www.prism.oregonstate.edu">PRISM</a>, and <a href="http://www.worldclim.org">WorldClim</a>.</p>

<p>This package is designed with the large-scale geographic information system (GIS) use-case in mind: cases where the use of dynamic web-services is impractical due to the scale (spatial and/or temporal) of analysis. It functions primarily as a means of downloading tiled or otherwise spaticially-defined datasets; additionally, it can preprocess those datasets by extracting data within an area of interest (AoI), defined spatially. It relies heavily on the <a href="http://cran.r-project.org/package=sp"><strong>sp</strong></a>, <a href="http://cran.r-project.org/package=raster"><strong>raster</strong></a>, and <a href="http://cran.r-project.org/package=rgdal"><strong>rgdal</strong></a> packages, and requires three command line tools be installed by the user (and accesible through <code>system()</code> calls in <em>R</em>): <a href="https://www.gnu.org/software/wget/">wget</a> (for downloading with timestamping), <a href="http://www.gdal.org">GDAL</a> (for manipulating raster and vector spatial data), and <a href="http://mdbtools.sourceforge.net">mdbtools</a> (for extracting data from access databases). I <strong>strongly</strong> recommend <a href="http://brew.sh">Homebrew</a> for installing <em>R</em> and each of these command-line tools.</p>

<p>This package has been built and tested on Mac OS 10.10 (Yosemite), and has been successfully run on an Ubuntu Linux cluster.</p>

<h3>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h3>

<p>To install, use the following command in <em>R</em>:</p>

<p><code>devtools::install_github("bocinsky/FedData")</code></p>

<p>A demo script is available in the <code>/inst</code> folder at the location of the installed package.</p>
