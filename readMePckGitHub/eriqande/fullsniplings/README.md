    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-fullsniplings" class="anchor" href="#fullsniplings" aria-hidden="true"><span class="octicon octicon-link"></span></a>fullsniplings</h1>

<p>An R package to infer full-sibling relationships from genetic data.
Geared toward datasets with many individuals and about 100 SNPs which are
treated as if they are unlinked. (While that is clearly false, it might be
reasonable).  The method is based on computing the posterior predictive
distribution of genotypes conditional on currently-inferred sibling groups. 
I will eventually generalize
that to more general pedigree inference, but for now just want to implement
the easy full-sibling case as a prototypical exercise.</p>

<h2>
<a id="user-content-installing--building" class="anchor" href="#installing--building" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installing / Building</h2>

<p>On a Mac you need to have the developer command line tools, and on a PC you will 
want the RTools to build this package.  You need to have the <code>Rcpp</code> package as
well, and also the <code>devtools</code> package.  You can get both of those from CRAN.</p>

<p>Here is what you would do:</p>

<div class="highlight highlight-r"><pre><span class="pl-c"># download, build and install the package</span>
<span class="pl-e">devtools</span><span class="pl-k">::</span>install_github(<span class="pl-v">repo</span><span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>fullsniplings<span class="pl-pds">"</span></span>, <span class="pl-v">username</span><span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>eriqande<span class="pl-pds">"</span></span>)</pre></div>

<h2>
<a id="user-content-running-a-simple-data-set" class="anchor" href="#running-a-simple-data-set" aria-hidden="true"><span class="octicon octicon-link"></span></a>Running a Simple Data set</h2>

<p>Take a look at the first few lines of a SNP data set that has 1157 Chinook typed at 95
SNPs:</p>

<div class="highlight highlight-r"><pre>head(<span class="pl-smi">fs_dev_test_data</span><span class="pl-k">$</span><span class="pl-smi">chinook_full_sibs_genos</span>)</pre></div>

<p>It is simple two column format (each locus gets two alleles) with the rownames being
the IDs of the fish.  Missing data are denote by NA, as is typical in R.</p>

<p>The main function for running the MCMC is <code>run_mcmc</code>.  It is not fully documented, yet, but
you can do:</p>

<div class="highlight highlight-r"><pre>?<span class="pl-smi">run_mcmc</span></pre></div>

<p>to learn about it.  The example shows how to do a short run.</p>

<p>If you want to do a longer run (about 5 minutes) you can do</p>

<div class="highlight highlight-r"><pre><span class="pl-smi">quick_run</span>  <span class="pl-k">&lt;-</span> run_mcmc(<span class="pl-smi">fs_dev_test_data</span><span class="pl-k">$</span><span class="pl-smi">chinook_full_sibs_genos</span>, <span class="pl-v">burn_in</span> <span class="pl-k">=</span> <span class="pl-c1">50</span>, <span class="pl-v">num_sweeps</span> <span class="pl-k">=</span> <span class="pl-c1">200</span>)</pre></div>

<h2>
<a id="user-content-building-a-vignette-that-shows-how-i-used-fullsniplings-for-my-talk-at-the-coastwide-meeting-in-june-of-2014" class="anchor" href="#building-a-vignette-that-shows-how-i-used-fullsniplings-for-my-talk-at-the-coastwide-meeting-in-june-of-2014" aria-hidden="true"><span class="octicon octicon-link"></span></a>Building a vignette that shows how I used fullsniplings for my talk at the Coastwide meeting in June of 2014</h2>

<p>To do this, you will need V2 of the <code>rmarkdown</code> package.  You can get that  either
by using the latest preview version of Rstudio: <a href="http://www.rstudio.com/products/rstudio/download/preview/">http://www.rstudio.com/products/rstudio/download/preview/</a>
or by installing rmarkdown from github like this:</p>

<div class="highlight highlight-r"><pre><span class="pl-e">devtools</span><span class="pl-k">::</span>install_github(<span class="pl-s"><span class="pl-pds">"</span>eriqande/fullsniplings<span class="pl-pds">"</span></span>)</pre></div>

<p>Then, once you have done that you should be able to do </p>

<div class="highlight highlight-r"><pre><span class="pl-e">rmarkdown</span><span class="pl-k">::</span>render(system.file(<span class="pl-s"><span class="pl-pds">"</span>dev_vignettes/full-sibling-chinook-mcmc.Rmd<span class="pl-pds">"</span></span>, <span class="pl-v">package</span><span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>fullsniplings<span class="pl-pds">"</span></span>), <span class="pl-v">output_format</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>html_document<span class="pl-pds">"</span></span>, <span class="pl-v">output_file</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>~/full-sibling-chinook-mcmc.html<span class="pl-pds">"</span></span>)</pre></div>

<p>That will put the output in your home directory in the file <code>full-sibling-chinook-mcmc.html</code> as long as your system
recognizes the <code>~</code>.  If not, just put an absolute path of your choice in for the <code>output_file</code> parameter in the 
<code>render</code> function.</p>

<p>Note that this doesn't run the MCMC, it just grabs some stored results.  But you can go into the code of </p>

<div class="highlight highlight-r"><pre>system.file(<span class="pl-s"><span class="pl-pds">"</span>dev_vignettes/full-sibling-chinook-mcmc.Rmd<span class="pl-pds">"</span></span>, <span class="pl-v">package</span><span class="pl-k">=</span><span class="pl-s"><span class="pl-pds">"</span>fullsniplings<span class="pl-pds">"</span></span>)</pre></div>

<p>and uncomment the appropriate lines to run it all.</p>
