    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-umx" class="anchor" href="#umx" aria-hidden="true"><span class="octicon octicon-link"></span></a>umx</h1>

<p>umx stands for "user" mx function, and is a comprehensive library of helper functions for doing <a href="http://en.wikipedia.org/wiki/Structural_equation_modeling">Structural Equation Modeling</a> in <a href="http://openmx.psyc.virginia.edu">OpenMx</a>.</p>

<p>You can install the most recent version from github as follows</p>

<div class="highlight highlight-splus"><pre>install.packages(<span class="pl-s"><span class="pl-pds">"</span>devtools<span class="pl-pds">"</span></span>)
library(<span class="pl-s"><span class="pl-pds">"</span>devtools<span class="pl-pds">"</span></span>)
install_github(<span class="pl-s"><span class="pl-pds">"</span>tbates/umx<span class="pl-pds">"</span></span>)
library(<span class="pl-s"><span class="pl-pds">"</span>umx<span class="pl-pds">"</span></span>)
?<span class="pl-smi">umx</span>
<span class="pl-c"># on windows you might need</span>
<span class="pl-c"># install_github("umx", username = "tbates", args = "--no-multiarch")</span>
<span class="pl-c"># On the newest version of devtools, this will work</span>
<span class="pl-c"># install_github("tbates/umx")</span></pre></div>

<p>The umx package has a wide range of functions (about 100) to get things done in SEM. The best way to learn about what is on offer is to install, and use the help, starting with '?umx'.
Some highlights include:</p>

<ol class="task-list">
<li>Building Models

<ul class="task-list">
<li>
<code>umxRAM()</code> <em># mxModel with an R-like <code>data =</code> parameter, no need to specify type = "RAM", + auto-discovery of manifests and latents from the paths you write</em>
</li>
<li>
<code>umxPath()</code> <em># mxPath on steroids: one-parameter settings for <code>var=</code> , <code>mean =</code> <code>cov=</code>, <code>fixedAt</code> and more. Great time-saver in RAM modelling</em>
</li>
<li>
<code>umxACE()</code> <em># 1-line twin modeling</em>
</li>
<li>
<code>umxStart()</code> <em># Add plausible start values to RAM or matrix models, or matrices: *</em>very** helpful*</li>
<li>
<code>umxLabel()</code> <em># Add systematic labels to paths ("var1_to_var2") and matrix cells ("a_r1c1"). This is great for setting, equating and dropping paths by label!</em>
</li>
</ul>
</li>
<li>Reporting output

<ul class="task-list">
<li>
<code>umxSummary(model)</code> # <em>Get a brief summary of model fit, similar to a journal report (Χ², p, CFI, TLI, &amp; RMSEA). Optionally show the path loadings</em>
</li>
<li>
<code>plot(model, std=TRUE, precision=3, dotFilename="name")</code> # <em>Create a graphical representation of models (outputs a <a href="http://www.graphviz.org/Gallery.php">GraphViz</a> file)</em>
</li>
<li>
<code>umx_time(fit1)</code>  <em># Report the time taken by a model in a compact programable format</em>
</li>
</ul>
</li>
<li>Modify models

<ul class="task-list">
<li>
<code>umxReRun()</code> *# Modify and run a model. You can add objects, drop or add paths, including by regular-expression label matching), re-name the model, re-run, and even return the comparison. All in 1 line *</li>
<li>
<code>umxGetParameters(model, regex = "as_r_2c_[0-9]", free = T)</code> <em># A powerful assistant to get labels from a model. like <code>omxGetParameters</code> but uses regular expressions.</em>
</li>
<li>
<code>umxMI()</code> <em># Report the top n modification indices</em>
</li>
<li>
<code>umxAdd1()</code> <em># add parameters and return a table of the effect on fit</em>
</li>
<li>
<code>umxDrop1()</code> <em># Drop parameters and return a table of the effect on fit</em>
</li>
</ul>
</li>
<li>Data and package helpers

<ul class="task-list">
<li>
<code>umxHcor(data, use = "pairwise.complete.obs")</code> <em># Compute appropriate pair-wise correlations for mixed data types.</em>
</li>
</ul>
</li>
</ol>

<p>Feel free to use, and improve: Log suggestions here using the Github comments, wiki, or git.</p>
