    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-binner" class="anchor" href="#binner" aria-hidden="true"><span class="octicon octicon-link"></span></a>binner</h1>

<p><code>binner</code> is an R package for processing DNA fingerprint data. At present, it
provides a complete workflow for AFLP analysis:</p>

<ol class="task-list">
<li>reading ABI <code>.fsa</code> files</li>
<li>normalizing electropherograms</li>
<li>identifying and sizing peaks</li>
<li>viewing individual electropherograms</li>
<li>dropping, adding and renaming samples</li>
<li>Automated peak-binning, using the <code>RawGeno</code> algorithm</li>
<li>Generating presence-absence matrices for further analysis in R (or
export for use in other programs, if you like)</li>
</ol>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>You can install <code>binner</code> using Hadley Wickham's
<a href="https://github.com/hadley/devtools">devtools</a> package.</p>

<pre><code>install.packages("devtools")
install_github("plantarum/binner")
</code></pre>

<h2>
<a id="user-content-help" class="anchor" href="#help" aria-hidden="true"><span class="octicon octicon-link"></span></a>Help</h2>

<p>There is no vignette for the package. I have provided a reasonably complete
example in the help file for the function <code>readFSA</code>. Open it from with R
with <code>?readFSA</code> (after you've loaded binner of course).</p>

<h2>
<a id="user-content-alternatives" class="anchor" href="#alternatives" aria-hidden="true"><span class="octicon octicon-link"></span></a>Alternatives</h2>

<p>There are two R-based alternatives you may like to consider as well:</p>

<ul class="task-list">
<li><a href="http://sourceforge.net/projects/rawgeno/">RawGeno</a></li>
<li><a href="https://r-forge.r-project.org/projects/aflp/">AFLP</a></li>
</ul>

<h3>
<a id="user-content-rawgeno" class="anchor" href="#rawgeno" aria-hidden="true"><span class="octicon octicon-link"></span></a><a href="http://sourceforge.net/projects/rawgeno/">RawGeno</a>
</h3>

<p><code>RawGeno</code> currently requires the use of a second program,
<a href="https://www.lifetechnologies.com/order/catalog/product/4381867">PeakScanner</a>,
to read the raw fsa data. <code>binner</code> is a bit slower than <code>PeakScanner</code> for
reading fsa files, but implements the same sizing algorithm (local
southern). Parts of <code>binner</code> may make their way into the next version of
<code>RawGeno</code>, and the current version of <code>binner</code> has pilfered some key bits of
<code>RawGeno</code> (most notably the binning algorithm).</p>

<p>If working entirely in R is useful to you, <code>binner</code> is currently preferable
to <code>RawGeno</code>. <code>binner</code> also provides a GUI for checking and editing your bins.
Beyond that it's a matter of preference.</p>

<h3>
<a id="user-content-aflp" class="anchor" href="#aflp" aria-hidden="true"><span class="octicon octicon-link"></span></a><a href="https://r-forge.r-project.org/projects/aflp/">AFLP</a>
</h3>

<p><code>AFLP</code> provides a more sophisticated workflow than <code>binner</code> or <code>RawGeno</code>, and
provides a very powerful system for controlling for variation between
capillaries, sequencers, sequencing runs etc. However, it seems to use a
different sizing system, making it hard to make direct comparisons between
electropherograms processed with <code>AFLP</code> and those using the
<code>PeakScanner</code>/<code>binner</code> approach. </p>
