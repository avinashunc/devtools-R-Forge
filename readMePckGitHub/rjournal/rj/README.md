    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-rj" class="anchor" href="#rj" aria-hidden="true"><span class="octicon octicon-link"></span></a>RJ</h1>

<p>This package provides tools for dealing with the metadata for the R Journal. The overall goal of the package is to automate common tasks performed by the editors, particularly the editor-in-chief.</p>

<h2>
<a id="user-content-description-format" class="anchor" href="#description-format" aria-hidden="true"><span class="octicon octicon-link"></span></a>
<code>DESCRIPTION</code> format</h2>

<p>Each submission consists of a directory containing the submission, any correspondence, and a <code>DESCRIPTION</code> file.  The <code>DESCRIPTION</code> is a DCF (debian control file) file, made up of entries of the following form.</p>

<pre><code>Title: Exploring the wonderful world of R
Authors: 
  "John Smith" &lt;john.smith@gmail.com&gt;
Editor: Hadley Wickham
Reviewers: 
  "Marleen Harris" &lt;marline.harris@hotmail.com&gt;, 
  "Roger Swansong" &lt;iloveswans52@yahoo.com&gt;
Status: 
  2012-05-04 submitted,
  2012-05-18 acknowledged [optional comment],
  2012-09-12 out for review,
  2012-10-12 minor revision,
  2012-11-12 accepted,
  2012-12-12 proofed,
  2013-01-01 published
</code></pre>

<p>Most of these fields are self-explanatory, apart from the <code>Status</code> field, which needs a little more detail. Some statuses can be automatically determined:</p>

<ul class="task-list">
<li>If editor field is blank, then status is "Needs editor"</li>
<li>If reviewers field is blank, then status is "Needs reviewers"</li>
</ul>

<p>See <code>valid_status</code> for a list of all known statuses, and <code>todo()</code> for the next action to be taken based on each status.</p>

<h2>
<a id="user-content-useful-actions" class="anchor" href="#useful-actions" aria-hidden="true"><span class="octicon octicon-link"></span></a>Useful actions</h2>

<div class="highlight highlight-R"><pre>library(<span class="pl-smi">rj</span>)
<span class="pl-c"># Make sure your working directory in the articles directory then:</span>

as.article(<span class="pl-s"><span class="pl-pds">"</span>2010-15<span class="pl-pds">"</span></span>)
new_id()
report()

<span class="pl-c"># Accept, reject or withdraw an article, by updating status, moving to</span>
<span class="pl-c"># correct directory and drafting emails.</span>
accept(<span class="pl-s"><span class="pl-pds">"</span>2010-15<span class="pl-pds">"</span></span>)
reject(<span class="pl-s"><span class="pl-pds">"</span>2010-15<span class="pl-pds">"</span></span>)
withdraw(<span class="pl-s"><span class="pl-pds">"</span>2010-15<span class="pl-pds">"</span></span>)
