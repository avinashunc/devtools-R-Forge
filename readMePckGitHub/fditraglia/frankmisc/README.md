    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-frankmisc" class="anchor" href="#frankmisc" aria-hidden="true"><span class="octicon octicon-link"></span></a>frankmisc</h1>

<p>An R package containing miscellaneous functions that I use all the time.</p>

<p>This is my first attempt to create an R package after many shameful years spent cutting and pasting. This is still a work in progress since I'm still a beginner at creating R packages, but it's not as difficult as I thought it would be. </p>

<h2>
<a id="user-content-references" class="anchor" href="#references" aria-hidden="true"><span class="octicon octicon-link"></span></a>References</h2>

<p>I was inspired to try this out after reading a post on <a href="http://hilaryparker.com/2014/04/29/writing-an-r-package-from-scratch/">Hilary Parker's blog</a>. Two other valuable resources for R package creation are Hadley Wickham's <a href="http://adv-r.had.co.nz/">Advanced R Programming</a> and the documentation for <a href="https://github.com/klutometis/roxygen#roxygen2">roxygen2</a>. One of these days I'm going to get around to reading the definitive documentation contained in <a href="http://cran.r-project.org/doc/manuals/R-exts.html#Creating-R-packages">Writing R Extensions</a>.</p>

<h2>
<a id="user-content-package-workflow" class="anchor" href="#package-workflow" aria-hidden="true"><span class="octicon octicon-link"></span></a>Package Workflow</h2>

<p>The basic workflow for creating and updating a package is as follows. If they're not installed already, install the packages <code>devtools</code> and <code>roxygen2</code> and then load them</p>

<pre><code>install.packages("devtools")
library("devtools")
devtools::install_github("klutometis/roxygen")
library(roxygen2)
</code></pre>

<p>To create a new package in a new directory do the following:</p>

<pre><code>setwd("parent_directory")
create("your_package_here")
</code></pre>

<p>This creates a number of files and directories. One of them is the R directory. Put your R code here, and see the <code>roxygen2</code> help files for how to add documentation comments. To process the documentation, go to the package directory and run the command <code>document()</code></p>

<pre><code>setwd("./your_package_here")
document()
</code></pre>

<p>To install the package, go the the <em>parent directory</em> and use the <code>install</code> command</p>

<pre><code>setwd("parent_directory")
install("your_package_here")
</code></pre>

<p>To install directly from a Github repo, do the following:</p>

<pre><code>install_github("your_package_here", "github_username")
</code></pre>

<p>For example, to install <code>frankmisc</code> make sure that <code>devtools</code> is loaded and then run</p>

<pre><code>install_github("frankmisc", "fditraglia")
</code></pre>

<p>Finally, use <code>library</code> as per usual to load the package.</p>

<pre><code>library(frankmisc)
</code></pre>
