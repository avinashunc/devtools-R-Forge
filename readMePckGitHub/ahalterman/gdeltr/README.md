    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-gdeltr" class="anchor" href="#gdeltr" aria-hidden="true"><span class="octicon octicon-link"></span></a>gdeltr</h1>

<p><code>gdeltr</code> is my ad-hoc collection of functions for working with <a href="http://gdelt.utdallas.edu/">GDELT</a>.  It is completely untested on any other machine, has no error catching functions, and has completely excessive dependencies on other packages.  I recommend copying and pasting any code you find useful rather than installing the whole package.</p>

<p>Two basic utilities for working with the traditional event stream:</p>

<ul class="task-list">
<li>
<code>gDate</code> for converting dates from a "yyymmdd" string to a "yyyy-mm-dd" Date class.</li>
<li>
<code>fillSeries</code> to add missing days to a GDELT data frame for plotting or time series analysis</li>
</ul>

<p>Two more advanced functions for pulling events from a <code>dplyr</code>/SQLite setup, as described <a href="http://andrewhalterman.com/2013/08/28/gdelt_dplyr_sqlite/">here</a>:</p>

<ul class="task-list">
<li>
<code>subsetEventCountry</code> for returning lat/long fields given a country name and EventRootCode.</li>
<li>
<code>getEventCounts</code> for returning base, root, or regular event codes per month for a given country.</li>
</ul>

<h3>
<a id="user-content-global-knowledge-graph" class="anchor" href="#global-knowledge-graph" aria-hidden="true"><span class="octicon octicon-link"></span></a>Global Knowledge Graph</h3>

<p>The Global Knowledge Graph is the newest component of GDELT.  In his <a href="http://gdeltblog.wordpress.com/2013/10/27/announcing-the-debut-of-the-gdelt-global-knowledge-graph/">announcement</a>, Kalev describes it as an attempt "to connect every person, organization, location, count, theme, news source, and event across the planet into a single massive network that captures what's happening around the world, what its context is and who's involved, and how the world is feeling about it, every single day."
He also points out that it's much more difficult to work with than the original event stream and recommends using Perl or Python for working with it.  (For one example, it's a nested structure using a combination of tabs, hashtags, and semicolons as separators.)  While Perl/Python may be better, there are lots of people (including me) who are much more comfortable working in R, even if it's inferior.   Here are my quick hacks for working with the alpha experimental release of GDELT's Global Knowledge Graph:</p>

<ul class="task-list">
<li>
<code>GKGcomentions</code> for pulling co-mentioned organizations, people, or countries from a subsetted GKG file.</li>
<li>
<code>GKGextractcameo</code> will return the events from the traditional stream associated with a subset of the GKG namespaces.  It can return either the vector of GLOBALEVENTIDs, or, if you have a dplyr/sqlite setup, the full data frame of events.</li>
<li>
<code>toner</code> will, for a given GKG subset, return the tones associated with each person/place/organization associated with it.</li>
<li>
<code>GKGcounts</code> will take a subset of the GKG and return just the info in the <code>COUNTS</code> column, nicely formatted.  This refers to info in the "Counts" column, not sums of number of events as above.</li>
<li>
<code>GKGedgelist</code> will take a ragged data frame with co-mentions and format it into a two-column edge list for export to Gephi or other network analysis tool.</li>
<li>
<code>write.gephi</code>: a wrapper for <code>write.table</code> that puts quotes around all elements in the df and writes with semicolon separators and without row/column names.</li>
<li>
<code>nameFixer</code> will standardize names from the GKG.  Only has about 30 (mostly Syria-related) names right now.  This isn't really worth using yet and I'm sure there's a better approach than this.</li>
<li>
<code>themeTrend</code>. This will plot the number of mentions of arbitrary themes per day.</li>
<li>
<code>toneTrend</code>. This will plot the average tone of coverage per day of persons, organizations, or themes.</li>
</ul>
