<h2>
<a id="user-content-generate-interactive-plots-in-aaaarrrrrrrr" class="anchor" href="#generate-interactive-plots-in-aaaarrrrrrrr" aria-hidden="true"><span class="octicon octicon-link"></span></a>generate interactive plots in AAAARRRRRRRR</h2>

<h1></h1>

<p><em><a href="http://www.csvsoundsystem.com/">by csv soundsystem</a></em></p>

<p>An R package to create interactive HighCharts in JavaScript from a dataframe. You'll want to use dcast in the reshape2 package to format it nicely. <a href="http://www.cookbook-r.com/Manipulating_data/Converting_data_between_wide_and_long_format/">More on that</a></p>

<p>Currently supported chart types: bar, line, area, column.</p><p>
</p><p>Chart types to come: datetime x-axis (coming soon), scatterplot and others</p>

<p><strong>Terms of service note: This library uses the HighCharts.js library. Which, in addition to being awesome, has some particular terms of use if you are a for-profit venture. Before using it on your site, we suggest <a href="http://shop.highsoft.com/highcharts.html" target="_blank">you make sure you are squared with their terms</a>.</strong></p>

<h2>
<a id="user-content-install-the-package" class="anchor" href="#install-the-package" aria-hidden="true"><span class="octicon octicon-link"></span></a>Install the package</h2>

<h2>
<a id="user-content-what-data-structure-does-it-need" class="anchor" href="#what-data-structure-does-it-need" aria-hidden="true"><span class="octicon octicon-link"></span></a>What data structure does it need?</h2>

<p>There are two general data formats when dealing with dataframes and we'll use the terminology "long" and "wide". For reference, read <a href="http://www.cookbook-r.com/Manipulating_data/Converting_data_between_wide_and_long_format/">the R Cookbook explanation</a> on the difference between the two. Generally, a long format is one where each row contains all the information for a given record so you can append records one at a time. </p>

<pre><code>name,variable,value
John,apples,2
John,oranges,3
Jane,pears,1
</code></pre>

<p>A wide format is one that is column dependent</p>

<pre><code>name,apples,oranges,pears
John,2,3,0
Jane,0,01
</code></pre>

<p>highchARRRts prefers your data to be in long format except for categorical data, like the example above.</p>

<p>To convert from long to wide, use the dcast method in the reshape2 package for R. The <a href="http://www.cookbook-r.com/Manipulating_data/Converting_data_between_wide_and_long_format/">R Cookpost post</a> can tell you about all the different parameters you might need to use depending on the complexity of your data. </p>

<p>For everything else, long format is what you'll want, which is great because your data is probably already in long format.</p>

<h2>
<a id="user-content-usage" class="anchor" href="#usage" aria-hidden="true"><span class="octicon octicon-link"></span></a>Usage</h2>

<p>highchARRRts has the following required parameters
...</p>

<pre><code>highchaRRRts(dcasted_df, type, main, xlab, ylab, output, pal)

</code></pre>

<p>Then in your console, cd to the directory it just created, default name "chart_output". Then create a simple web server such as</p>

<pre><code>python -m SimpleHTTPServer
</code></pre>

<p>Then point your browser to 0.0.0.0:8000</p>

install_github("highChaRRRts", "csvsoundsystem")
library("highChaRRRts")

# generate example data...

var1 &lt;- rnorm(11, mean=100, sd=30)

df &lt;- data.frame(group, var1, var2, var3, var4, var5)

             xlab="X Label",
             ylab="Y Label",
             output = "chart_output",
