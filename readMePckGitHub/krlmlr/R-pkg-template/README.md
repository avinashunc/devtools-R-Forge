    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-r-package-template" class="anchor" href="#r-package-template" aria-hidden="true"><span class="octicon octicon-link"></span></a>R package template</h1>

<p>This is a template for an empty R package with <code>roxygen2</code> and RStudio support.
The following gets you started with a fresh R package on a Linux system
(requires <a href="http://defunkt.io/hub/"><code>hub</code></a>).
The package will be pushed to GitHub automatically:</p>

<pre><code>NEW_NAME=YourNewPackage
git init $NEW_NAME
cd $NEW_NAME
hub remote add krlmlr/R-pkg-template
git remote rename krlmlr R-pkg-template
git pull --all
hub create $NEW_NAME
git pull --ff R-pkg-template stable-0.6
tools/rename $NEW_NAME
git push -u origin master
</code></pre>

<p>Additional features might include:</p>

<ul class="task-list">
<li>a <code>Makefile</code>
</li>
<li>...</li>
</ul>

<p>The idea is to keep the <code>stable-*</code> branches stable so that they can be pulled
from even for existing R packages. Moving from one stable branch to another
might require some effort.</p>

<h2>
<a id="user-content-travis-ci-integration" class="anchor" href="#travis-ci-integration" aria-hidden="true"><span class="octicon octicon-link"></span></a>Travis-CI integration</h2>

<p>With <a href="https://travis-ci.org/">Travis-CI</a> you can have your package checked
automatically each time you push to GitHub. All you need is to login and
flip a switch on the Travis-CI website,
the service is free of charge for open-source projects.
The template already contains a generic <code>.travis.yml</code> that connects to
<a href="https://github.com/craigcitro/r-travis">r-travis</a>.</p>

<h2>
<a id="user-content-pushing-to-r-forges-subversion-server" class="anchor" href="#pushing-to-r-forges-subversion-server" aria-hidden="true"><span class="octicon octicon-link"></span></a>Pushing to R-Forge's Subversion server</h2>

<p>The <code>tools</code> subdirectory contains two scripts that allows setting up a <code>git-svn</code>
connection to R-Forge (or any other SVN server, for that matter). The setup
closely follows the <a href="http://cameron.bracken.bz/git-with-r-forge">blog post</a>
by Cameron Bracken.</p>

<p>To initialize the link, execute <code>tools/setup-r-forge &lt;your-svn-URL&gt;</code>. If you
link with R-Forge, this should be the the path to the <code>pkg</code> subdirectory in the
SVN repo, or to the <code>pkg/pkg-name</code> subdirectory if there are several packages.
When executed the first time, all previous Git commits in the <code>master</code> branch
are pushed as a single
commit to SVN. The script should also work for a fresh Git checkout when
the SVN repo contains (a part of) the commits in <code>master</code>. In this case,
for technical reasons, sometimes a dummy commit will be pushed to SVN.
After that, <code>tools/update-r-forge</code> will push the current state of <code>master</code> to
SVN.</p>

<p>Both scripts require a clean working copy. Should you ever need to merge changes
from SVN back to Git, you can execute <code>git merge r-forge</code> for the
<code>master</code> branch.</p>
