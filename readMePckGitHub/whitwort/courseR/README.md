    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-courser" class="anchor" href="#courser" aria-hidden="true"><span class="octicon octicon-link"></span></a>courseR</h1>

<p>courseR is a tool chain designed to build websites from <a href="https://github.com/rstudio/rmarkdown">R Markdown</a> sources.  This third (and hopefully final) rewrite now leverages the power of R Markdown v2, which in turn uses <a href="http://yihui.name/knitr/">Knitr</a> and <a href="http://johnmacfarlane.net/pandoc/">Pandoc</a> to do the heavy lifting during page rendering.  While originally designed to produce websites for courses that are taught primarily in R, the package is likely to be generally useful when building any sort of website from a set of RMarkdown sources.  The chief features are:</p>

<ol class="task-list">
<li><p>Source files can be RMarkdown (.rmd), plain markdown (.md), or R scripts.  The courseR headers are now fully compatible with RMarkdown v2.0 headers, making it easy to do <em>ad hoc</em> page renders with RMarkdown (which will come packaged in future versions of RStudio).</p></li>
<li><p>Single source files can specify multiple output destinations; for example, in the default project, single .RMD sources are used to produce handout-style HTML documents, PDFs, and HTML slides.</p></li>
<li><p>Separation in the build workflow between an intermediate page rendering step and the final website building step to allow templates to make use of site-wide information (for example, to build global tables of contents).</p></li>
<li><p>Smart(ish) caches between builds.  The default Knitr caching mechanism caches the results of chunk blocks, re-runing blocks if the code in them changes.  Unfortunately, there's no way to automatically invalidate these caches when external files they depend upon change.  Instead of using R object chunk-caches, courseR caches the each output that is rendered by RMarkdown.  If source files or data they depend upon change, all associated output files are re-rendered; if templates or configuration settings for a particular output target change, only these files are re-rendered.</p></li>
</ol>

<h3>
<a id="user-content-installing" class="anchor" href="#installing" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installing</h3>

<p>courseR and RMarkdown v2.0 are both on github, so the easiest way to install them is to use the devtools package.  If you don't have it already:</p>

<div class="highlight highlight-S"><pre>install.packages(<span class="pl-s"><span class="pl-pds">"</span>devtools<span class="pl-pds">"</span></span>)</pre></div>

<p>To install courseR and the new version of RMarkdown from github:</p>

<div class="highlight highlight-S"><pre>devtools::install_github(<span class="pl-s"><span class="pl-pds">"</span>rstudio/rmarkdown<span class="pl-pds">"</span></span>)
devtools::install_github(<span class="pl-s"><span class="pl-pds">"</span>whitwort/courseR<span class="pl-pds">"</span></span>)</pre></div>

<h3>
<a id="user-content-starting-a-project" class="anchor" href="#starting-a-project" aria-hidden="true"><span class="octicon octicon-link"></span></a>Starting a project</h3>

<p>To initialize a new template project directory either start a blank project in RStudio or manually set your working directory into an empty path, then call:</p>

<div class="highlight highlight-S"><pre>library(courseR)
courseR.init()</pre></div>

<p>See the new README.md file in the root of the project folder for a full description of the structure of the subfolders that were created and the courseR.yaml file for example configuration options.  When you're ready to build the project website, just call:</p>

<div class="highlight highlight-S"><pre>courseR.build()</pre></div>
