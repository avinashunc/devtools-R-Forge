<a id="user-content-solarsatellite-geolocation-for-animal-tracking" class="anchor" href="#solarsatellite-geolocation-for-animal-tracking" aria-hidden="true"><span class="octicon octicon-link"></span></a>Solar/Satellite Geolocation for Animal Tracking</h1>

<p>SGAT (pronounced "tags backwards") provides facilities for estimating
broadscale animal motions from archival or satellite tag data.</p>

<h2>
<a id="user-content-installing" class="anchor" href="#installing" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installing</h2>

<p>The package is easily built with RStudio</p>

<ol class="task-list">
<li><p>Install R</p></li>
<li><p>Install <a href="http://www.rstudio.com">RStudio</a></p></li>
<li><p>Install <a href="http://cran.r-project.org/bin/windows/Rtools/">Rtools</a> or equivalent for your platform</p></li>
<li><p>Install <a href="http://cran.r-project.org/web/packages/devtools/index.html">devtools</a> and <a href="http://cran.r-project.org/web/packages/roxygen2/index.html">roxygen2</a> packages and dependencies in R</p></li>
<li><p>Install <a href="http://cran.r-project.org/web/packages/geosphere/index.html">geosphere</a>, <a href="http://cran.r-project.org/web/packages/GeoLight/index.html">GeoLight</a>, <a href="http://cran.r-project.org/web/packages/sp/index.html">sp</a>, <a href="http://cran.r-project.org/web/packages/raster/index.html">raster</a>, <a href="http://cran.r-project.org/web/packages/maptools/index.html">maptools</a> packages and dependencies.</p></li>
<li><p>Clone the repository from GitHub (<a href="https://github.com/SWotherspoon/SGAT">https://github.com/SWotherspoon/SGAT</a>).</p></li>
<li><p>Create an Rstudio project in the folder containing this README file.</p></li>
<li><p>In the build tab, choose <code>More/Configure Build Tools...</code> and click
<code>Generate documentation with Roxygen</code>, select <code>Configure</code> and choose to generate <code>Rd files</code> and the <code>NAMESPACE file</code>, leaving the other options as they are.</p></li>
<li><p>Choose <code>More/Roxygenize</code> from the <code>Build</code> tab</p></li>
<li><p>Choose <code>Build &amp; Reload</code> to make the package immediately available to R, or choose <code>More/Build source package</code> <code>More/Build binary package</code> from the <code>Build</code> tab to make source or binary packages.</p></li>
</ol>

<h2>
<a id="user-content-todo" class="anchor" href="#todo" aria-hidden="true"><span class="octicon octicon-link"></span></a>TODO</h2>

<ul class="task-list">
<li><p><strong>Alternate Coordinates</strong>.  The Metropolis sampler may be more efficient if locations are represented in geocentric ecliptic coordinates. But it is unclear whether the gains in efficiency due to higher acceptance rates would outweigh the costs of the coordinate transformation and the added code complexity. This is a low priority.</p></li>
<li><p><strong>Parallelisation</strong>.  At this point, the Metropolis samplers are only capable of utilizing a single core on a multicore machine.  It would be relatively simple to introduce coarse grain parallelism by having the samplers draw multiple chains in parallel, using something like the multicore facility in the parallel package.  Unfortunately, at the time of writing there does not seem to be a good parallization solution that works equally well on all platforms.  This is a low priority.</p></li>
</ul>
