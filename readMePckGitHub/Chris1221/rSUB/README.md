    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-rsub-painless-qsub-for-r-scripts" class="anchor" href="#rsub-painless-qsub-for-r-scripts" aria-hidden="true"><span class="octicon octicon-link"></span></a>rSUB: Painless qSUB for R Scripts</h1>

<p><strong>Author:</strong> Christopher Cole</p>

<p><strong>Version:</strong> 0.1</p>

<p><strong>Description:</strong> With a little bit of set up, submitting cluster jobs should become easy and quick.  rSUB submits jobs to an SGE cluster looped around several variables (2 at the moment). These will need to be specified by you in your R script priod to use. The function will do all the legwork to submit your script with each unique combination of the variables provided.</p>

<h3>
<a id="user-content-using-rsub" class="anchor" href="#using-rsub" aria-hidden="true"><span class="octicon octicon-link"></span></a>Using rSUB</h3>

<p>This package submits seperate jobs for each unique combination of every variable inputted. Specify in your R script where you would like these to be placed, either in the reading of a file name, or in the calculation of a variable.  The name you give will be edited out and replaced with command args notation in the background.  Remember the name that you specified, as it will be needed for the function later on.  For this example, we will be using the variable name DOGNAME to individually submit jobs based on three levels, namely Spot, Rover, and Killer.  In the R script <em>MyDogName.R</em> we will specify the place that these variables will take.</p>

<pre><code>print("My favourite dog is", DOGNAME)
</code></pre>

<p>Note that DOGNAME as a variable must be outside of any quotation marks in order to retain its function as a variable.</p>

<p>The second thing that we need to set up beforehand is a config file for the cluster that you are personally using.  Here is mine called "config.txt": </p>

<pre><code>#!/bin/bash
#$ -S /bin/bash
#$ -q abaqus.q
#$ -l qname=abaqus.q
#$ -cwd
#$ -V
#$ -j y
#$ -o /home/hpcxxxx/logs/$JOB_ID.txt
</code></pre>

<p>For more information on how to construct a cluster config file, [please see here])<a href="http://www.hpcvl.org/faqs/general/how-to-run-programs-grid-engine">http://www.hpcvl.org/faqs/general/how-to-run-programs-grid-engine</a>). </p>

<p>Now that we have our inital set up, we can launch right into submiting to the cluster. </p>

<p>Install rSUB (binaries coming soon), go to your working directory containing your script and config file, and initialize the following values.</p>

<pre><code>github_install("rSUB", "Chris1221")
library("rSUB")
rscript &lt;- MyDogName.R #this is the script we will be submitting
cluster_header &lt;- config.txt #this is our cluster config
arg1_name &lt;- DOGNAME #this is the same name that we gave our variable in the script
arg1_values &lt;- c("Spot", "Rover", "Killer") #the levels upon which we will submit.  
</code></pre>

<p>That's it!  Let's submit this baby.</p>

<pre><code>rsub(rscript, cluster_header, arg1_name, arg1_values)
</code></pre>

<p>And it's as simple as that! No more messing around with .sge files and slave/master structures. </p>
