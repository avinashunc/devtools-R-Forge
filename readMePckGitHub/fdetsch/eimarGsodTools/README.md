    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-eimargsodtools" class="anchor" href="#eimargsodtools" aria-hidden="true"><span class="octicon octicon-link"></span></a>eimarGsodTools</h1>

<h3>
<a id="user-content-what-it-is-all-about" class="anchor" href="#what-it-is-all-about" aria-hidden="true"><span class="octicon octicon-link"></span></a>What it is all about</h3>

<p>Every person dealing with long-term climatological data (e.g. of daily air
temperature, relative humidity, and precipitation amounts) will sooner or later
stumble across the Global Summary Of Day (GSOD) climate data collection 
provided by the National Oceanic and Atmospheric Association (NOAA). In the 
course of my PhD thesis, I've been recently looking for available GSOD 
stations in close vicinity to Mt. Kilimanjaro, Tanzania, and as I am trying to
realize most of my coding work using R, I quickly noticed that there are only a 
few packages that provide convenient tools for processing GSOD data. Therefore, 
I started to write this package that includes both downloading data sets of 
selected climate stations for a given time span as well as 
some processing steps (detection of statistic outliers, gap-filling based upon
linear interpolation, linear models and singular spectrum analysis (SSA)) for 
quality assurance.</p>

<h3>
<a id="user-content-introducing-the-processing-chain" class="anchor" href="#introducing-the-processing-chain" aria-hidden="true"><span class="octicon octicon-link"></span></a>Introducing the processing chain</h3>

<p>I am currently struggling with loading sample data automatically into the R
workspace. Any kind of advice on that would be highly appreciated. The ASCII file 
containing the information about all the single GSOD stations around the world 
is absolutely necessary in order to find a desired location for data download, 
and right now, I see now other way than importing it manually by calling <code>data()</code>.</p>

<p><strong>Getting started</strong></p>

<p>After loading the package and the corresponding data, I thought it might be quite
useful to optionally sort out some inconveniences in the GSOD station list. The
referring function is called <em>gsodReformat()</em>. The function includes the possibilities
to reformat the following columns:</p>

<ul class="task-list">
<li>elevation: initially in dm, transferred to m if <code>elevation = TRUE</code> (default)</li>
<li>coordinates: initially in decimal degrees / 1000, transferred to 'usual' 
decimal degrees if <code>coords = TRUE</code> (default); in addition, all invalid coordinates, 
i.e. longitude &lt; -180° or &gt; 180° and latitude &lt; -90° or &gt; 90° degree, are 
repliced with NA.</li>
<li>the supplied data.frame can be converted into a <code>SpatialPointsDataFrame</code> if
<code>df2sp = TRUE</code> (default is <code>df2sp = FALSE</code>)</li>
</ul>

<p>Hence, the first lines of code using <strong>eimarGsodTools</strong> should probably look like this:</p>

<div class="highlight highlight-S"><pre><span class="pl-c"># Load package and related data</span>
library(eimarGsodTools)
data(gsodstations)

<span class="pl-c"># Reformat data and convert to spatial object</span>
stations.<span class="pl-k">sp</span> &lt;- gsodReformat(data = gsodstations,
                            elevation = TRUE, 
                            coords = TRUE,
                            df2sp = TRUE)</pre></div>
