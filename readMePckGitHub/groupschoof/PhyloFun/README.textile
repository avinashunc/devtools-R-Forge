    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-phylofun" class="anchor" href="#phylofun" aria-hidden="true"><span class="octicon octicon-link"></span></a>PhyloFun</h1>
<h2>
<a id="user-content-abstract" class="anchor" href="#abstract" aria-hidden="true"><span class="octicon octicon-link"></span></a>Abstract</h2>
<p>Protein function has often been transferred from characterized proteins to novel proteins based on sequence similarity, e.g. using the best BLAST hit. Based on the SIFTER phylogenomic tool (1), we use a statistical inference algorithm to propagate e.g. Gene Ontology (GO) terms inside a phylogenetic tree, scoring branch lengths and evidence codes of GO annotations. Here PhyloFun computes the likelihood of a GO term being inherited on a given phylogenetic branch based on probability distributions that have been carefully calibrated for each GO term separately.</p>
<p>In order to generate accurate phylogenetic trees that contain a maximum of functional information at reasonable computational costs, we implemented a reusable workflow that, for a given input protein, searches candidate orthologs with known functions, adds paralogs so that duplications can be detected reliably and builds a maximum likelihood phylogenetic tree from a filtered multiple alignment. This tree is then used as input to the inference algorithm which outputs, for each protein in the tree, a probability for assigning each GO term occurring in the tree.</p>
<p>We call this new phylogenomic workflow for protein function prediction <strong>PhyloFun</strong>.</p>
<h2>
<a id="user-content-install-phylofun" class="anchor" href="#install-phylofun" aria-hidden="true"><span class="octicon octicon-link"></span></a>Install PhyloFun</h2>
<p>PhyloFun requires <code>R</code> version <code>2.15.2</code> <em>or greater</em>.</p>
<h3>
<a id="user-content-phylofun-requires-three-external-tools" class="anchor" href="#phylofun-requires-three-external-tools" aria-hidden="true"><span class="octicon octicon-link"></span></a>PhyloFun requires three external tools:</h3>
<ul>
	<li>MAFFT <a href="http://mafft.cbrc.jp/alignment/software/">http://mafft.cbrc.jp/alignment/software/</a>
</li>
	<li>GBlocks <a href="http://molevol.cmima.csic.es/castresana/Gblocks.html">http://molevol.cmima.csic.es/castresana/Gblocks.html</a>
</li>
	<li>FastTree[MP] <a href="http://www.microbesonline.org/fasttree/">http://www.microbesonline.org/fasttree/</a>
</li>
</ul>
<p>Make sure all three programms are installed and in your search path. That means, that from an interactive <code>R</code> shell the following must work:<br>
<code>system( "mafft" )</code><br>
<code>system( "GBlocks" )</code><br>
<code>system( "FastTree" )</code> or, if you installed the preferred multi processor (OpenMP) version of FastTree:<br>
<code>system( "FastTreeMP" )</code></p>
<h3>
<a id="user-content-install-required-r-packages" class="anchor" href="#install-required-r-packages" aria-hidden="true"><span class="octicon octicon-link"></span></a>Install required R packages:</h3>
<p>From within R execute the followin code to install the R packages PhyloFun requires.<br>
</p><pre><code>source("http://bioconductor.org/biocLite.R")
biocLite( c(
  "Rcpp", "Biostrings", "RCurl", "RMySQL", "XML", "ape", "biomaRt", "brew",
  "gRain", "phangorn", "rredis", "stringr", "xtable"
) )</code></pre>
<h3>
<a id="user-content-install-the-phylofun-r-package-itself" class="anchor" href="#install-the-phylofun-r-package-itself" aria-hidden="true"><span class="octicon octicon-link"></span></a>Install the PhyloFun R package itself:</h3>
<ul>
	<li>Download source: <code>git clone git://github.com/groupschoof/PhyloFun.git ./PhyloFun</code>
</li>
	<li>Install: <code>R CMD INSTALL PhyloFun</code>
</li>
</ul>
<h2>
<a id="user-content-run-phylofun" class="anchor" href="#run-phylofun" aria-hidden="true"><span class="octicon octicon-link"></span></a>Run PhyloFun</h2>
<p>Please <em>note</em> that PhyloFun requires a working internet connection to run properly!</p>
<p>The PhyloFun R package comes with a number of executable Rscripts, all stored in folder <code>exec/</code>.</p>
<p>After installation find the path to the installed PhyloFun package. Open an interactive <code>R</code> shell and type <code>.path.package("PhyloFun")</code>.<br>
The returned path can than be used to run the provided Rscript <code>runPhyloFun.R</code> as follows:</p>
<p><code>Rscript &lt;path_to_your_PhyloFun_installation&gt;/exec/runPhyloFun.R &lt;arguments&gt;</code></p>
<h3>
<a id="user-content-phylofun-command-line-arguments" class="anchor" href="#phylofun-command-line-arguments" aria-hidden="true"><span class="octicon octicon-link"></span></a>PhyloFun command line arguments</h3>
<p>The arguments are printed whenever the Rscript is executed.</p>
<ul>
	<li>
<code>-q</code> path to Query Proteins’ amino acid sequences in fasta format</li>
	<li>
<code>-p</code> or <code>-b</code> path to sequence similarity search results. Provide <code>-b</code> for BLAST tables (tabular output <code>-m 8</code>) or <code>-p</code> for PHMMER (HMMER-3) search results in <code>--tblout</code> format.</li>
	<li>
<code>-e</code> provide comma separated list of GO evidence codes you want to be accepted as “trusted”, e.g. <code>-e ISO,RCA</code> or provide <code>-e ALL</code> if any evidence code is good for you. See <a href="http://www.geneontology.org/GO.evidence.shtml">http://www.geneontology.org/GO.evidence.shtml</a> for details on GO evidence codes.</li>
	<li>
<code>-n</code> use the n best hits for function prediction. If you want to speed up PhyloFun, reduce this number to e.g. <code>-n 250</code>
</li>
	<li>
<code>-h true</code> to print out statistics of each Query Proteins’ found homologs – result file <code>homologs_stats.txt</code>
</li>
	<li>
<code>-m true</code> to print out statistics of each generated Multiple Sequnce Alignment (MSA) – result file <code>msa_stats.txt</code>
</li>
	<li>
<code>-r true</code> to generate an HTML report of PhyloFun’s results</li>
</ul>
<h3>
<a id="user-content-a-short-test-run-can-be-done-as-follows" class="anchor" href="#a-short-test-run-can-be-done-as-follows" aria-hidden="true"><span class="octicon octicon-link"></span></a>A short test run can be done as follows</h3>
<pre><code>Rscript &lt;path_to_your_PhyloFun_installation&gt;/exec/runPhyloFun.R -q &lt;path_to_your_PhyloFun_installation&gt;/protein_1.fasta -b &lt;path_to_your_PhyloFun_installation&gt;/protein_1_blastout.tbl -f FastTree -h true -r true -m true</code></pre>
<p><em>Note</em> that depending on you installation of FastTree you’ll have to provide either <code>-f FastTree</code> or the faster multi processor version of if <code>-f FastTreeMP</code>.</p>
<p>Results of this test run will be written into the folder <code>Protein_1/</code> in your current directory.</p>
<h3>
<a id="user-content-generate-phylofuns-input" class="anchor" href="#generate-phylofuns-input" aria-hidden="true"><span class="octicon octicon-link"></span></a>Generate PhyloFun’s input</h3>
<p>The only thing you have to do, is to run <em>either</em> <strong>BLAST</strong> or <strong>PHMMER</strong> for your query proteins against the provided database of UniprotKB proteins with trusted Gene Ontology term annotations (<strong>trusted-UniKB</strong>).</p>
<ol>
	<li>First copy and unpack <strong>trusted-UniKB</strong> from PhyloFun to your directory of choice. <code>cp inst/ukb_proteins_with_trusted_go_annos.fasta.bz2 path/to/your/directory</code>, and then unpack it <code>bunzip2 ukb_proteins_with_trusted_go_annos.fasta.bz2</code>
</li>
	<li>Now run BLAST or PHMMER. If you choose BLAST you must provide its output as a table, use the command line switch <code>-m 8</code>. If you choose PHMMER a tabular result file will also be required and can be obtained by usage of the command line argument <code>--tblout</code>.</li>
	<li>Recommended E-Value cutoffs for BLAST are <code>-e 1</code> and for PHMMER <code>-E 0.000001</code>
</li>
</ol>
<h2>
<a id="user-content-coding-style-guide" class="anchor" href="#coding-style-guide" aria-hidden="true"><span class="octicon octicon-link"></span></a>Coding style guide</h2>
<p>PhyloFun’s code mostly follows the google style guide for R:<br>
<a href="http://google-styleguide.googlecode.com/svn/trunk/google-r-style.html">http://google-styleguide.googlecode.com/svn/trunk/google-r-style.html</a></p></article>
