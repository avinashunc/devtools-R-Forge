    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-rrd" class="anchor" href="#rrd" aria-hidden="true"><span class="octicon octicon-link"></span></a>Rrd</h1>

<p>R package for working with <a href="http://oss.oetiker.ch/rrdtool/">RRD</a> files. Uses <a href="http://oss.oetiker.ch/rrdtool/doc/librrd.en.html">librrd</a> to import the binary data directly into R without exporting it to an intermediate format first.</p>

<h1>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h1>

<p>In order to build the package from source you need <a href="http://oss.oetiker.ch/rrdtool/doc/librrd.en.html">librrd</a>.<br>
Installing <a href="http://oss.oetiker.ch/rrdtool/">RRDtool</a> from your package manager will usually also install the library.  </p>

<p>In the parent directory of Rrd:  </p>

<pre><code>R CMD build Rrd
R CMD check Rrd
sudo R CMD INSTALL Rrd
</code></pre>

<h1>
<a id="user-content-usage" class="anchor" href="#usage" aria-hidden="true"><span class="octicon octicon-link"></span></a>Usage</h1>

<p>In R:</p>

<pre><code>library(Rrd)
</code></pre>

<h2>
<a id="user-content-importrrdfilename-consolidation-function-start-end-step" class="anchor" href="#importrrdfilename-consolidation-function-start-end-step" aria-hidden="true"><span class="octicon octicon-link"></span></a>importRRD(filename, consolidation function, start, end, step)</h2>

<p>Finds the RRA that best matches the <strong>consolidation function</strong> and the <strong>step</strong> and imports all values (from all data stores) in that RRA that are between timestamp <strong>start</strong> and <strong>end</strong>. <strong>start</strong> is not included in the result.</p>

<pre><code>rra = importRRD("filename.rrd", "AVERAGE", 1402144275, 1402144410, 15)
</code></pre>

<p>Returns a data.frame object having the timestamp and the data stores as columns. The data store names are retrieved from the RRD file and set as the corresponding column names. The timestamps are also used as row names.</p>

<pre><code>head(rra)

            timestamp sum num
1402144275 1402144275   0   0
1402144290 1402144290   0   0
1402144305 1402144305   0   0
1402144320 1402144320   0   0

<h2>
<a id="user-content-importrrdfilename" class="anchor" href="#importrrdfilename" aria-hidden="true"><span class="octicon octicon-link"></span></a>importRRD(filename)</h2>

<p>Retrieves the metadata from the RRD file and imports all values from this file - each RRA, each data store, from the first to the last timestamp. </p>

<pre><code>rrd = importRRD("filename.rrd")
</code></pre>

<p>Returns a list of data.frames (as above) with labels constructed as "consolidation function" + "step" - e.g. "AVERAGE15".</p>

<h2>
<a id="user-content-getvalfilename-consolidation-function-step-timestamp" class="anchor" href="#getvalfilename-consolidation-function-step-timestamp" aria-hidden="true"><span class="octicon octicon-link"></span></a>getVal(filename, consolidation function, step, timestamp)</h2>

<p>Returns a data.frame row associated with a certain <strong>timestamp</strong>. Reads small "chunks" of the RRA (see the <strong>rrd.cacheBlock</strong> constant in source code) and uses a package-wide read-ahead cache implemented as an <a href="http://stat.ethz.ch/R-manual/R-devel/library/base/html/environment.html">environment</a> object. The <strong>timestamp</strong> and <strong>step</strong> arguments are also used to calculate the row index in the cache to ensure fast indexing (as opposed to looking up by row name, which is known to be much slower in R). The cache is guaranteed to store a maximum of <strong>rrd.cacheSize</strong> rows per RRA.</p>

<pre><code>getVal("filename", "AVERAGE", 15, 1402231740)

            timestamp sum num
1402231740 1402231740   0   0


rrd.cache$`filenameAVERAGE15`
</code></pre>

<p>For more information on rrdtool and the rrd format please refer to the official rrdtool <a href="http://oss.oetiker.ch/rrdtool/doc/index.en.html">documentation</a> and <a href="http://oss.oetiker.ch/rrdtool/tut/index.en.html">tutorials</a>.</p>

<p>A more in-depth description of the package and more examples can be seen <a href="http://plamendimitrov.net/blog/2014/08/09/r-package-for-working-with-rrd-files/">here</a>.</p>
