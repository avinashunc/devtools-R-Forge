<p>Dear relenium users, </p>

<p>We have noticed that other Selenium bindings have been developed and successfully contributed on CRAN (<a href="https://github.com/ropensci/RSelenium">RSelenium</a>). Aleix and me (Lluís) think that it doesn't make much sense to have two Selenium bindings in the R community. As we have failed to contribute <a href="https://github.com/LluisRamon/relenium">relenium</a> and <a href="https://github.com/LluisRamon/seleniumJars">seleniumJars</a> on CRAN, we encourage you to switch to RSelenium. We will be updating Selenium JARS for a while but we will no longer improve relenium or fix any issues, neither try again to contribute it on CRAN. We thank all of you for the comments, improvements and ideas that have been shared with us. We have enjoyed developing relenium and we are happy that a Selenium binding is available in the R community. RSelenium is an <a href="http://ropensci.org/packages/">rOpenSci</a> package and we are sure that it will be well maintained.</p>
</blockquote>

<hr>

<h2>
<a id="user-content-introduction" class="anchor" href="#introduction" aria-hidden="true"><span class="octicon octicon-link"></span></a>Introduction</h2>

<p><a href="http://docs.seleniumhq.org/">Selenium</a> is a popular suite of tools specifically designed for automating web browsers. Selenium allows users to write tests in a number of popular programming languages including Java. The <strong>relenium package</strong> is intended to provide access to Selenium and use it to boost web navigation and to ease the path for web scraping in R.</p>

<h2>
<a id="user-content-development" class="anchor" href="#development" aria-hidden="true"><span class="octicon octicon-link"></span></a>Development</h2>

<p>Some members of the <a href="http://rugbcn.wordpress.com/">Barcelona R user group</a> are developing relenium. Right now, relenium is in beta testing. Some errors might be found and documentation is still been improved.</p>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>You can install <code>relenium</code> from <code>github</code> using the <code>devtools</code> package. A recent version of Java Runtime Environment (JRE) and Firefox web browser are required. <a href="https://github.com/LluisRamon/seleniumJars">seleniumJars</a> and <a href="http://cran.r-project.org/web/packages/rJava/index.html">rJava</a> R packages are also needed. You can get <code>rJava</code> from <code>CRAN</code> and <code>seleniumJars</code> from <code>github</code>.</p>

<pre><code>install.packages("rJava")

require(devtools)

install_github('seleniumJars', 'LluisRamon')
install_github('relenium', 'LluisRamon')
</code></pre>

<p>In case you are using Windows you will also need <a href="http://cran.r-project.org/bin/windows/Rtools/">rtools</a>. In case R is trying to access to the Intenet through a proxy, follow this <a href="http://stackoverflow.com/questions/17783686/solution-how-to-install-github-when-there-is-a-proxy">SO question</a>. If you have trubles with the installation, it is suggested to reinstall Java, rJava and relenium. Check that all of them, including R, correspond to the same version (32 or 64 bits). </p>

<h2>
<a id="user-content-bugs-and-new-features" class="anchor" href="#bugs-and-new-features" aria-hidden="true"><span class="octicon octicon-link"></span></a>Bugs and new features</h2>

<p>To report technical problems with relenium, please file an issue on <a href="https://github.com/LluisRamon/relenium/issues">Github</a> with a <a href="http://adv-r.had.co.nz/Reproducibility.html">short reproducible example</a>. Including the ouptut of the following code will make much easier to track any problem.</p>

<pre><code>library(relenium)

sessionInfo()

firefox &lt;- firefoxClass$new()
firefox$getCapabilities()
firefox$getVersion()
firefox$close()
</code></pre>

<p>Feedback is welcomed and any new features should be filed as issues as well.</p>
