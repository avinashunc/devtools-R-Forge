    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-r-helpers" class="anchor" href="#r-helpers" aria-hidden="true"><span class="octicon octicon-link"></span></a>R-helpers</h1>

<p>R-helpers is a jumble of functions that I load by default when I start up R (to see how this is done, look at .Rprofile).  I have not packaged them as an R library.  If you see anything that you want to use, just copy the file, and source it.  Some of them require having external libraries installed (which you can see by looking for calls to <code>library</code>).  If this still doesn't work, there may be dependencies on other R-helper files, so you might want to copy everything and try again. :-p</p>

<p>I have tried to make the files work independently of each other, and document the functions, but I am sure there are omissions and bugs.</p>

<p>I hope that R-helpers can be useful to other R users.  Here are some highlights:</p>

<ul class="task-list">
<li><p>inspect.R: I expect this will be the most useful function here.  In the process of debugging, it is common for R programmers to write code like: <code>cat("x = "); print(x)</code>  but this is repetitive and inflexible.  <code>inspect</code> handles objects that can be printed inline (e.g. integers and strings) as well as objects that can't be (e.g. matrices, functions).  More generally, it handles R expressions, including function calls.  Furthermore, we make it easy to turn off all calls to inspect in one go (by setting <code>NO_INSPECT</code>), or to turn on only a subset of inspects (by setting <code>ACTIVE_TAGS</code>).</p></li>
<li><p>plotting.R: <code>pplot</code> is useful for debugging when you have overlapping plots.  It figures out when to call <code>plot</code> vs <code>points</code>.</p></li>
<li><p>plotting.R: There is also some code for generating color functions (i.e. functions from [0,1] to RGB colors) from RColorBrewer palettes as well as pairs of colors.</p></li>
<li><p>plotting.R: There is a function for changing the background color, <code>colorBackground</code>.  Must be used before plotting other elements.</p></li>
<li><p>string.R: function to compute the Levenshtein edit distance between two strings or two character vectors.</p></li>
<li><p>meta-programming.R: for any distribution following the R naming convention, <code>distributionFuns</code> wraps together the <code>p</code>,<code>q</code>,<code>r</code>,<code>d</code> functions (i.e. cdf, quantile, random sample, pdf) as "methods" of a Distribution object.  This is one place in R where a little Object-Orientation would be welcome, and <code>distributionFuns</code> is one step in that direction.</p></li>
<li><p>meta-programming.R: I have a reimplementation of <code>with</code>, called <code>getProperty</code>.</p></li>
</ul>
