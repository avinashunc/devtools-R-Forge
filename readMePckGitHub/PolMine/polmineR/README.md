<h2>
<a id="user-content-base-package-of-the-polmine-toolkit" class="anchor" href="#base-package-of-the-polmine-toolkit" aria-hidden="true"><span class="octicon octicon-link"></span></a>Base package of the PolMine-Toolkit</h2>

<h3>
<a id="user-content-purpose" class="anchor" href="#purpose" aria-hidden="true"><span class="octicon octicon-link"></span></a>Purpose</h3>

<p>The purpose of the package 'polmineR' is to facilitate the interactive analysis of corpora using R. Core objectives for the development of the package are performance, usability, and a modular design.</p>

<p>There are many tools already for text mining. Why yet another one? Important incentives for developing the package were:</p>

<ul class="task-list">
<li>to create a package that makes the creation and analysis of subcorpora (called 'partitions' here) as easy as possible. A particular strength of the package should be to support contrastive/comparative research.</li>
<li>to keep the original text accessible. The polmineR is based on the conviction that statistical analysis alone may be blind and deaf.</li>
<li>to provide an open source platform that will make text mining more productive, avoiding prohibitive costs of any kind. Well, some familiarity with R is still necessary.</li>
</ul>

<h3>
<a id="user-content-design" class="anchor" href="#design" aria-hidden="true"><span class="octicon octicon-link"></span></a>Design</h3>

<p>The polmineR relies on the <em>Open Corpus Workbench</em> (CWB) as a backend and uses the <em>rcqp</em> package as an interface. The CWB is particularly efficient for storing large corpora and offers a powerful language for querying corpora, the Corpus Query Processor (CQP). The architecture may be overengineered if you work with smaller corpora. It is meant to make working with larger corpora efficient, both locally, or on a server.</p>

<h3>
<a id="user-content-background" class="anchor" href="#background" aria-hidden="true"><span class="octicon octicon-link"></span></a>Background</h3>

<p>The polmineR-package was specifically developed to make full use of the XML annotation structure of the corpora created in the PolMine project (see polmine.sowi.uni-due.de). The core PolMine corpora are corpora of plenary protocols. In these corpora, speakers, parties etc. are structurally annotated. The polmineR-package is meant to help making full use of the rich annotation structure.</p>

<h3>
<a id="user-content-core-functions" class="anchor" href="#core-functions" aria-hidden="true"><span class="octicon octicon-link"></span></a>Core functions</h3>

<ul class="task-list">
<li>
<em>partition</em>: Set up a partition (i.e. subcorpus);</li>
<li>
<em>context</em>: Analyse the context of a query (including some statistics);</li>
<li>
<em>dispersion</em>: Analyse the dispersion of a query across one or two dimensions (absolute and relative frequencies);</li>
<li>
<em>keyness</em>: Compare two partitions to identify specific vocabulary (using a chi-square test).</li>
</ul>

<h3>
<a id="user-content-state-of-affairs" class="anchor" href="#state-of-affairs" aria-hidden="true"><span class="octicon octicon-link"></span></a>State of affairs</h3>

<p>There are quite a few further functions, some of which are experimental. The publication of the polmineR-package on CRAN is planned as soon as the portability of the package is ensured. Most recent developments will be available here on GitHub.</p>

<h3>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h3>

<p>Theoretically, it sould be easy to install the package with the <em>devtools</em> mechanism. It has been checked on a preliminary basis that the package is portable, but feedback is most welcome.
The tricky part of the installation will usually be the <em>rcqp</em> package. See the package vignette for some advice.</p>

<h3>
<a id="user-content-feedback" class="anchor" href="#feedback" aria-hidden="true"><span class="octicon octicon-link"></span></a>Feedback</h3>

<p>Getting feedback is most welcome! I want this to be a useful package not just for me. Please do get in touch: Andreas Blaette, University of Duisburg-Essen (<a href="mailto:andreas.blaette@uni-due.de">andreas.blaette@uni-due.de</a>).</p>
