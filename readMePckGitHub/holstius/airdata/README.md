    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-readme" class="anchor" href="#readme" aria-hidden="true"><span class="octicon octicon-link"></span></a>README</h1>

<p>Use this package to import official U.S. air quality data from <a href="https://ofmext.epa.gov/AQDMRS/aqdmrs.html" title="AQDMRS">AirData</a> into R. Seeking comment!</p>

<h2>
<a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>Install directly from GitHub using <a href="https://github.com/hadley/devtools" title="devtools">devtools</a>:</p>

<pre><code>if (!require("devtools")) install.packages("devtools")
library("devtools")
install_github("airdata", "holstius")
</code></pre>

<h2>
<a id="user-content-example-usage" class="anchor" href="#example-usage" aria-hidden="true"><span class="octicon octicon-link"></span></a>Example usage</h2>

<p>A typical query might start with a pollutant of interest. Here's how to identify the <em>parameter code</em> for such a pollutant:</p>

<pre><code># Fetch all existing parameter codes
params &lt;- AQDMRS.list(name="param")

# Find just the one(s) corresponding to a search phrase
library(stringr)
phrase &lt;- "black carbon"
show(subset(params, str_detect(tolower(name), phrase)))
</code></pre>

<p>There are two such codes. <code>LC (88313)</code> means "local conditions" and <code>STP (84313)</code> means "standard temperature and pressure". You can use either, although a particular site might record data for only one of the two. In this case, <code>STP</code> is what we want.</p>

<p>Before executing a query, you'll need an <a href="http://www.epa.gov/airdata/tas_Data_Mart_Registration.html">AirData username and password</a>. Then you can do the following:</p>

<pre><code># If you don't supply user="..." and pw="...", you'll be prompted
q &lt;- AQDMRS.query(
    state = "06",           # California
    county = "001",         # Alameda County
    site = "0011",          # West Oakland site
    bdate = "2012-04-15",   
    edate = "2012-04-23",
    param = "84313"
)
BC_STP &lt;- eval(q)
</code></pre>

<p>Queries are <em>lazy</em> and can me modified before being evaluated, or recycled after evaluation. This is handy, because you don't have to type common parameters more than once:</p>

<pre><code>q$param &lt;- "88313"
BC_LC &lt;- eval(q)
</code></pre>

<p>And, if you like, you can specify <code>user=...</code> and <code>pw=...</code> in the query, rather than entering them interactively.</p>

<p>A quick time-series plot:</p>

<pre><code>with(BC_STP, plot(GMT, value, main="West Oakland BC", ylab="ug/m3 (STP)"))
</code></pre>

<h2>
<a id="user-content-windows-support" class="anchor" href="#windows-support" aria-hidden="true"><span class="octicon octicon-link"></span></a>Windows support</h2>

<p>AirData has an annoying habit of appending <code>END OF FILE</code> to every dataset. On unix-compatible platforms, including OS X, it's fast and easy to eliminate this with a quick call to <code>grep.</code> On Windows, there's no builtin <code>grep</code>, and there's no fast workaround in R (AFAIK, but please comment if you know of one). A workaround (and you can do this on other platforms, too) is:</p>

<ol class="task-list">
<li>download a <code>.txt</code> file from the <a href="https://ofmext.epa.gov/AQDMRS/aqdmrs.html" title="AQDMRS">AirData</a> website (choose "DMCSV" for the format)</li>
<li>open it up in a text editor, delete the "END OF FILE" line, and re-save</li>
</ol>

<p>Then you can import the file into R with <code>read.DMCSV()</code>:</p>

<pre><code>library(airdata)
downloaded_file &lt;- file.choose() 
BC_STP &lt;- read.DMCSV(downloaded_file, omit.EOF=FALSE)
</code></pre>
