    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h2>
<a id="user-content-data-analysis-menu-in-shiny" class="anchor" href="#data-analysis-menu-in-shiny" aria-hidden="true"><span class="octicon octicon-link"></span></a>Data Analysis Menu in Shiny</h2>

<p>This is a data-analysis menu for R using <a href="http://www.rstudio.com/shiny/">shiny</a>. If you would like to see the app in action please visit Rstudio's Glimmer-server at <a href="http://glimmer.rstudio.com/mostlyhrmls/damshiny/">http://glimmer.rstudio.com/mostlyhrmls/damshiny/</a></p>

<h3>
<a id="user-content-todo" class="anchor" href="#todo" aria-hidden="true"><span class="octicon octicon-link"></span></a>Todo:</h3>

<ul class="task-list">
<li>Add analysis-tools (e.g., MDS, factor, perceptual maps, conjoint)</li>
<li>Create help files</li>
<li>Add options for compare means (e.g., one-sided) and regression (e.g., F-test for competing models)</li>
<li>Add time-series plots with 1 or 2 y-axes</li>
<li>Develop install structure so users can pick-and-choose menu-components. For an amazing implementation see <a href="http://twitter.github.com/bootstrap/customize.html">Bootstrap</a> although this is <em>well</em> beyond what I have in mind.</li>
<li>etc. etc.</li>
</ul>

<h3>
<a id="user-content-to-use-the-damshiny-package-locally" class="anchor" href="#to-use-the-damshiny-package-locally" aria-hidden="true"><span class="octicon octicon-link"></span></a>To use the damshiny package locally</h3>

<pre><code>install.packages('shiny')
install.packages('damshiny', repos = c('http://vnijs.rady.ucsd.edu/site_media/R_packages/','http://cran.rtudio.com'))
library(shiny)
shiny::runApp(system.file('damshiny',package='damshiny'))
</code></pre>

<h3>
<a id="user-content-to-use-the-damshiny-development-version-locally" class="anchor" href="#to-use-the-damshiny-development-version-locally" aria-hidden="true"><span class="octicon octicon-link"></span></a>To use the damshiny development version locally</h3>
