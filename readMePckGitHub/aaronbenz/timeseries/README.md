    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a id="user-content-timeseries" class="anchor" href="#timeseries" aria-hidden="true"><span class="octicon octicon-link"></span></a>timeseries</h1>

<p>This is a work-in-progress R package that will help you calculate and plot time-series data at great speed. </p>

<p>10 Million points can be processed in ~300 ms, reducing it down to only the critical points required to conceptually view your data.</p>

<h2>
<a id="user-content-to-download" class="anchor" href="#to-download" aria-hidden="true"><span class="octicon octicon-link"></span></a>To Download</h2>

<p>Install package devtools if not installed</p>

<div class="highlight highlight-r"><pre>install.packages(<span class="pl-s"><span class="pl-pds">"</span>devtools<span class="pl-pds">"</span></span>)</pre></div>

<p>Load devtools and Download timeseriesr package</p>

<div class="highlight highlight-r"><pre>library(<span class="pl-smi">devtools</span>)
install_github(<span class="pl-s"><span class="pl-pds">"</span>aaronbenz/timeseries<span class="pl-pds">"</span></span>)</pre></div>

<h2>
<a id="user-content-why-use-the-timeseriesr-package" class="anchor" href="#why-use-the-timeseriesr-package" aria-hidden="true"><span class="octicon octicon-link"></span></a>Why use the timeseriesr package?</h2>

<p>The goal of this package is to provide you with common time-series use cases while using more popular data types like data.frames/data.tables, as opposed to ts/its/irts/timeSeries/ti/mts/zoo/xts, because honestly, who has time for that when data.frames are so easy to understand. Nonetheless, I do like what xts package has been able to do and I recommend investigating it if you are about to dive deep into time-series analytics.</p>

<p>In the future, I will create a vignette to help you understand how to use this package, but for now, let me introduce you to the main function/algorithm that has inspired this work. </p>

<p><code>dtreduce</code> and <code>vreduce</code> give you the ability to plot millions of points almost instantly by drastically reducing the total amount of points. In the so-called internet of things, we often encounter time-stamped sensor data sets that consist of more points than we have pixels, and there is no reason to plot millions of points in order to understand what is going on. Let's take a simple example using a sine curve:</p>

<div class="highlight highlight-r"><pre><span class="pl-c">###WARNING: DO NOT TRY TO PRINT/PLOT THIS</span>
<span class="pl-smi">df</span> <span class="pl-k">&lt;-</span> <span class="pl-k">data.frame</span>(<span class="pl-v">time</span> <span class="pl-k">=</span> <span class="pl-c1">1</span><span class="pl-k">:</span><span class="pl-c1">5e6</span>, <span class="pl-v">value</span> <span class="pl-k">=</span> sin(seq(<span class="pl-c1">1</span>,<span class="pl-c1">10</span>, <span class="pl-v">length.out</span> <span class="pl-k">=</span> <span class="pl-c1">5e6</span>)))</pre></div>

<p>This data set is about 60 MB. If you tried to do a <code>plot(df)</code>, you would find yourself waiting for minutes to view this plot, and that's if you're lucky to escape without R crashing. Really, just unacceptable.</p>

<p>You could try a sampling method to get 1 out of every 10 points, and in this case, it would work. However, you would ultimately be unconfident in your result because you could very likely be skipping critical spikes.</p>

<p><code>dtreduce</code> solves this problem by gathering points of significance given a tolerance that you choose. So, for example:</p>

<div class="highlight highlight-r"><pre><span class="pl-smi">df_reduced</span> <span class="pl-k">&lt;-</span> dtreduce(<span class="pl-smi">df</span>, <span class="pl-v">column</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>value<span class="pl-pds">"</span></span>)
library(<span class="pl-smi">ggplot2</span>)
ggplot(<span class="pl-smi">df_reduced</span>, aes(<span class="pl-smi">time</span>, <span class="pl-smi">value</span>)) <span class="pl-k">+</span> geom_step()
<span class="pl-c">#OR</span>
plot(<span class="pl-smi">df_reduced</span>)</pre></div>

<p>Note that visualizing 5 million points with accuracy only took about 190 milliseconds!!!</p>

<div class="highlight highlight-r"><pre>library(<span class="pl-smi">microbenchmark</span>)
microbenchmark(plot(dtreduce(<span class="pl-smi">df</span>,<span class="pl-s"><span class="pl-pds">"</span>value<span class="pl-pds">"</span></span>)))
<span class="pl-c">#Unit: milliseconds</span>
